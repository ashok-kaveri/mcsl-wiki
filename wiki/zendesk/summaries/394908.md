# Ticket #394908 — PostNord label generation fails due to SEK-to-EUR currency conversion on zero-total orders

- **Customer**: Oscar Goldring (store-5yrznpmrvr.mybigcommerce.com)
- **Duration**: 2026-06-12 – ongoing

## Timeline & Key Phases

### Phase 1: Initial report and investigation (2026-06-12)
- Customer reported PostNord "Mypack Collect" service unavailable; order #172 shows "FAILURE Contact support" with empty request body (Comment #1)
- L1 requested and received permission to test label generation on order #172 (Comment #3, #4)
- Customer confirmed the same shipment works directly in PostNord portal using "Parcel Locker" (rebranded Mypack Collect), with screenshots (Comment #8)
- Escalated to L3 — rates not returned on Order Summary page, label generation failing (Comment #10)

### Phase 2: Order total investigation (2026-06-12 – 2026-06-17)
- L3 discovered order #172 and all international orders have order total of 0 in the app (Comment #12)
- L1 asked customer about zero totals; customer explained they run B2B with separate invoicing (Comment #13, #15)
- Customer granted BigCommerce admin access for deeper investigation (Comment #17, #19)

### Phase 3: Root cause identified and workaround applied (2026-06-23)
- L3 identified root cause: currency conversion from SEK to EUR fails when order total is 0 (division-by-zero or invalid conversion) (Comment #23)
- Temporary workaround: change "Preferred Shipment Currency" to "Order Currency" in app settings to bypass the conversion (Comment #23)
- L3 applied the workaround on the customer's store; orders 171 and 172 now in Processing status (Comment #25)
- Customer acknowledged the update (Comment #26)

## Open Issues

1. **Currency conversion bug: SEK-to-EUR fails on zero-total orders** — When order total is 0 and preferred shipment currency differs from order currency, the rate/label flow breaks (likely division by zero or invalid conversion input). Fix planned for next release. Blocked: dev team. Severity: high — blocks all international label generation for this store and any store with zero-total orders + currency mismatch. Area: label-generation. (Comment #23)

## Resolved Issues

1. **Orders 171 and 172 stuck in failure state** — Workaround applied by changing Preferred Shipment Currency to Order Currency; orders moved to Processing. (Comment #25)
