# Ticket #384891 — FedEx duties payment configuration reset after migration causing unexpected invoices

- **Customer**: Filip Van Moerkercke (havensurf.myshopify.com)
- **Product**: Shopify Multi Carrier Shipping Label App
- **Duration**: 2026-04-20 to ongoing (3 days)

## Timeline & Key Phases

### Phase 1: Initial Report (2026-04-20)
- Customer reports receiving unexpected FedEx duty invoices during Multi Carrier app testing
- Customer's business model: Shipper pays shipping, recipients pay all duties/customs for non-EU orders
- Support investigates and identifies Duties Payor Type incorrectly set to "Sender" instead of "Recipient"
- Root cause: Setting was reset during FedEx REST API migration completed by customer

### Phase 2: Technical Investigation (2026-04-20 - 2026-04-21)
- Support confirms Multi Carrier app has correct "Recipient" setting
- Additional investigation reveals PH Ship Rate and Track for FedEx app also affected
- Customer had migrated from FedEx SOAP to REST API (ticket #379238 reference)
- Migration steps shared by support did not explicitly warn that duty payment settings would reset
- Customer completed migration independently without re-validating all settings
- Support manually corrects both apps to "Recipient" on 2026-04-20

### Phase 3: Customer Escalation (2026-04-21 - 2026-04-22)
- Customer escalates: 2 weeks of incorrect duty charges accumulated (FedEx invoices delayed)
- Requests reimbursement for unexpected costs
- Customer Success Manager (Keerthi) acknowledges insufficient communication about settings reset
- Offers 6-month complimentary subscription as goodwill gesture
- Clarifies PluginHive cannot intervene with FedEx billing directly

## Open Issues
