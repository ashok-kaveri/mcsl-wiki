## Open Issues

1. **EX Number Field Missing from WSS Plugin DG Configuration** — The plugin has no dedicated field for EX number (Explosive Classification), which is mandatory for shipping Class 1.4 Explosive primers with UPS HazMat. Customer cannot include it on generated DG labels. The UPS WooCommerce plugin already has a "HazMat Additional Description" field that serves this purpose; parity is needed in WSS. Blocked: customer (workaround in place but incomplete compliance). Severity: dev_needed tag, L3 escalated twice, customer is live and shipping without full compliance. Area: label-generation. (Comment #78, #80)

2. **UPS API Error "Technical Name Must Not Match Proper Shipping Name" Has No Plugin-Level Workaround** — UPS API rejects requests where the DG technical name exactly matches the proper shipping name. This is a real-world scenario for certain commodities (e.g., primers). The plugin surfaces the raw UPS error without guidance or a field-level validation to prevent it. Resolution path (engage uoltects@ups.com or modify the technical name) is manual and not documented in-app. Blocked: customer when this case recurs. Severity: recurring friction for HazMat users, no in-product resolution. Area: label-generation. (Comment #86, #87)

## Resolved Issues

1. **Checkout Real-Time Rates Overriding Existing Flat-Rate Shipping Rules** — After plugin installation, UPS live rates appeared at checkout instead of the store's flat rate ($12.95 / free over $150). Resolved by customer disabling the Realtime Rates option; support confirmed this is the correct configuration for stores not intending to show live rates at checkout. (Comment #37, #39)

2. **UPS Account B370W3 Not Eligible for HazMat Shipments** — HazMat label generation failed with API error "Shipper is not eligible to ship HazMat" for the primary UPS account. Resolved by identifying and adding the customer's dedicated HazMat UPS account (W68K87) to the plugin. (Comment #50, #53, #55)

3. **Orders Showing Incorrectly After Bulk "Exclude from Shipping" Update** — After bulk-importing products with "exclude from shipping" flag, existing orders did not reflect the update. Resolved by support refreshing the orders from the backend; customer taught to use "Reprocess Order" for future occurrences. (Comment #33, #34, #36)

4. **HazMat Label Not Generating Due to Incorrect DG Field Configuration** — Multiple label failures caused by incorrect product DG details (invalid Class Division entry, incorrect technical name, missing fields). Resolved iteratively via support calls and direct field corrections by the support team. Label successfully generated for orders 91066 and 91809. (Comment #64, #74)

5. **DG Documents Not Visible After Label Generation** — Customer could not find the HazMat dangerous goods documents after label creation. Resolved by support clarifying that DG documents appear as "Extra Documents" in the label view, not alongside the shipping label. (Comment #70)

6. **Emergency Contact Number on DG Document Not Configurable** — Customer wanted to update the emergency contact on the DG form. Resolved by support directing them to update the phone number in Plugin > Settings > Address (Ship From Address). (Comment #71, #72)

7. **UPS CTP Subsidy Approval and Account Activation** — Customer had a UPS Digital Connections subsidy code expiring May 15 but needing activation. PluginHive raised the CTP request with UPS; invoice of $696 confirmed processed. Customer installed plugin and CTP plan activated. (Comment #16, #25)
