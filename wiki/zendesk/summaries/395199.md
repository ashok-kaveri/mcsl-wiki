# Ticket #395199 — Commercial Invoice Shows Incorrect Freight Cost After Shopify Order Shipping Edit

- **Customer**: Wako Chapurlat (majo-garn.myshopify.com)
- **Duration**: 2026-06-15 to 2026-06-29 (ongoing)

## Timeline & Key Phases

### Phase 1: Initial Report — Order Total Mismatch (2026-06-15)
- Customer reported order #12125 (South Korea shipment) showing €3,612.87 in MCSL app instead of correct €3,128.00
- Root cause: shipping cost edited in Shopify after order import, but MCSL did not reflect the update
- Agent (Sagar) manually re-imported the order to fix the dashboard display
- L3 enabled Auto Order Updates feature for the store

### Phase 2: Commercial Invoice Bug Discovered (2026-06-16)
- Dashboard showed correct total after re-import, but commercial invoice still showed doubled shipping cost (€878.87 instead of €394)
- Customer canceled and regenerated label — problem persisted on commercial invoice
- Customer fulfilled order #12125 manually via FedEx website due to time pressure
- Same issue confirmed on order #12161: shipping edited multiple times (€530.79 → €479 → €508), but invoice showed sum of all entries (€1,517.79)
- Escalated to L3 — identified that Shopify retains all historical shipping entries in `total_shipping_price_set` and app sums all of them instead of using only the current active value (Comment #25, #27)

### Phase 3: Customer Test Order & Investigation (2026-06-17 to 2026-06-22)
- Customer created test order #12226 to reproduce the bug
- Confirmed: deleted €52.29 shipping + added €100 = invoice incorrectly showed €152.29
- Agent (Touseef) requested and received Edit Order permissions on the store
- Tested from PluginHive side and confirmed the issue in the Shopify order JSON

### Phase 4: Fix Identified, Enhancement Planned (2026-06-26 to 2026-06-29)
- L3 (Veena) contacted Shopify to understand `total_shipping_price_set` behavior (Comment #49)
- Solution identified: app enhancement to avoid summing historical shipping entries
- No timeline confirmed yet (Comment #50)
- Customer acknowledged and asked to cancel test order #12226 — approved (Comment #52)

## Open Issues

1. **Commercial invoice sums all historical Shopify shipping entries instead of current value** — When shipping cost is edited in Shopify (delete + re-add), the app reads `total_shipping_price_set` which accumulates all historical entries. Commercial invoice freight charges are overstated, causing incorrect customs declarations. Fix identified but not yet deployed. Blocked: dev team. Severity: critical (customs compliance risk, forced manual FedEx workaround). Area: label-generation. (Comment #50)

## Resolved Issues

1. **Order total not reflecting Shopify edits in MCSL dashboard** — Resolved by enabling Auto Order Updates feature and manual re-import. (Comment #11)
2. **Packing summary resets on order update** — Confirmed as unsupported; manual packing assignments are lost when order syncs. Not a bug, documented limitation. (Comment #11)
