# Ticket #376463 — Carrier name missing from BigCommerce shipment emails

- **Customer**: Kennedy Wild (UK-based BigCommerce store)
- **Product**: BigCommerce Multi Carrier Shipping Label App
- **Duration**: 2026-03-01 to 2026-03-05 (5 days)

## Timeline & Key Phases

### Phase 1: Initial Report (2026-03-01 to 2026-03-02)
- Customer reported shipment emails include tracking number but NOT carrier name
- Support requested screenshot to investigate (Comment #1-3)

### Phase 2: Investigation & Additional Requests (2026-03-03)
- Customer provided email screenshot (Comment #4)
- Support requested BigCommerce admin access to review order notification settings (Comment #6)
- Customer raised **two additional requests** while granting access:
  1. How to add carrier notes for delivery (e.g., customer order notes passed to carrier)
  2. How to set default delivery instructions like "Leave in safe place" (Comment #7)

### Phase 3: Access & Root Cause Analysis (2026-03-03 to 2026-03-04)
- Support confirmed BigCommerce sends order completion emails (not the app)
- App only adds tracking number to BigCommerce order; carrier name should be pulled from order data by BigCommerce
- Support clarified:
  - Order notes automatically pulled to label (25 char limit from ParcelForce API, not app)
  - Default delivery instructions feature not available (Comment #12-14)
- L3 escalation: Check if carrier name (`shipping_provider`) is being passed to BigCommerce fulfillment API (Comment #15)

### Phase 4: L3 Fix Identified (2026-03-05)
- **L3 root cause**: `shipping_provider` field not being passed in fulfillment request for ParcelForce
- Fix scheduled for next release (Comment #17-18)

## Open Issues

1. **ParcelForce carrier name not displayed in shipment emails** — BigCommerce order completion emails show tracking number but not "ParcelForce" carrier name. Root cause: App not passing `shipping_provider` field in fulfillment API request for ParcelForce shipments. Fix identified by L3, scheduled for next release. Blocked: customer (customers cannot identify carrier from email alone). Severity: Medium (usability/customer experience issue; workaround is customers check tracking URL). Area: `carrier-config`. (Comment #15, #17)

2. **No default delivery instructions option** — Customer requested ability to set default delivery instructions (e.g., "Leave in safe place") when order note is empty. L3 confirmed feature not available and cannot be configured. Blocked: customer (requires manual order notes for every shipment). Severity: Low (workaround is manual notes; affects operational efficiency). Area: `feature-request`. (Comment #14-15)
