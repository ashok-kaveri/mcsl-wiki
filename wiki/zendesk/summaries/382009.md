# Ticket #382009 — PO Box automation rule broken after FedEx REST migration; box dimensions missing on labels

- **Customer**: Zulay Bolton / Kona Coffee and Tea (a94ce6-2.myshopify.com)
- **Duration**: 2026-04-01 -- 2026-04-07 (ongoing)

## Timeline & Key Phases

### Phase 1: PO Box rule & dimensions issue reported (2026-04-01)
- Customer completed FedEx REST migration via Zoom call the previous day. Reported PO Box automation rule no longer auto-assigning USPS for PO Box orders (order #118028 was assigned FedEx instead). (Comment #0)
- Agent stated the rule was working but the order used a non-app checkout service, so cheapest carrier was auto-assigned. Customer disagreed, saying it worked correctly before the migration. (Comment #2, #3, #5)
- Customer also reported box dimensions no longer showing on FedEx labels since the REST migration. (Comment #5)

### Phase 2: Zoom call & investigation (2026-04-02 -- 2026-04-03)
- Zoom call conducted to discuss both issues. (Comment #11)
- L3 escalation raised for missing dimensions: FedEx REST API does not pass dimensions when a FedEx box is selected, unlike the old Webservice API which did. (Comment #12)

### Phase 3: FedEx dimensions resolution attempts (2026-04-06 -- 2026-04-07)
- L3 stated FedEx advised not to pass dimensions for FedEx boxes. (Comment #13, #20)
- Agent suggested using Custom Packing Slip as alternative. Customer pointed out the packing slip does not show selected package size, and the menu paths given were incorrect. (Comment #21, #23)
- Internal note: inconsistency found -- the SRT FedEx app does pass dimensions for FedEx boxes in REST API, but MCSL app does not. Team needs clarity on why this differs before providing a final answer. (Comment #25)

## Open Issues

1. **FedEx box dimensions not printed on labels after REST migration** -- FedEx REST API reportedly does not require dimensions for FedEx boxes, but the separate SRT FedEx app still passes them. Inconsistency under investigation. Customer needs dimensions for warehouse fulfillment workflow. (Comment #19, #25)
2. **PO Box automation rule behavior after migration** -- Customer claims the rule no longer auto-assigns USPS for PO Box orders as it did before the FedEx REST migration. Agent said the rule is working, but the root cause of changed behavior may not be fully investigated. (Comment #3, #5)
3. **Packing slip does not show selected package size** -- Suggested as workaround for missing label dimensions, but it does not include the auto-selected box info. Menu paths provided were incorrect for the customer's UI version. (Comment #23)

## Resolved Issues

None fully resolved.
