# Ticket #387640 — Postnord customer-selected service point overridden by default-SP automation rule

- **Customer**: Oskari / VILENIUS (Maustemesta, Finland) — oskari@maustemesta.com
- **Product**: Shopify Multi Carrier Shipping Label App
- **Duration**: 2026-05-05 to 2026-05-11 (6 days, ongoing — `dev_needed`)

## Timeline & Key Phases

### Phase 1: Initial Diagnosis & Misdiagnosis Loop (2026-05-05)
- Customer reports: when customer chooses a Postnord pickup location at checkout, the integration ignores it and still picks the closest one (#1).
- Internal triage: Finland store, 125 orders since 2025-09-11, PostNord-only, 29 labels in May, $29 plan (#2).
- Root cause identified: an automation rule with action **"Set Default Service Point for PostNord"** overrides the customer's selection at label-generation time (#3).
- Touseef (L1) advises customer to remove that automation rule action (#4).
- Customer asks: will it still pick the closest SP when no customer choice is made? Sibi confirms yes — that's the documented fallback (#5, #6).

### Phase 2: Side-effect Discovery — Removing Rule Broke Fallback (2026-05-08)
- Customer reports: after following the advice, **all orders now fail to create a label** (#7).
- Customer clarifies: when no service point is selected at checkout, the integration no longer falls back to the closest SP either (#8).
- Sibling ticket #388358 ("If the customer does not select a pickup location…") merged into this one — same regression (#9).
- Manohar V re-enables the default-service-point automation rule, restoring label generation for the no-customer-choice case, and asks customer to share a specific order where customer-selected SP was overridden so logs can be inspected. Requests Shopify collaborator access (sent to oskari@maustemesta.com) (#10).

### Phase 3: Reproduction & L3 Escalation (2026-05-08 to 2026-05-11)
- Customer provides example order **#33295**: customer chose **"PostNord Lähiboksi Kauppakeskus Lauttis 1"** but the app picked **"PostNord K-market Kasinonranta"** instead (#11).
- Manohar asks customer not to fulfill the order until reviewed; suggests using PostNord website directly as a manual workaround (#12).
- L3 escalation raised internally (#13) — explicit expected behavior documented:
  - Order with no SP selected → app auto-selects nearest SP (current rule's fallback). ✅
  - Order with customer-selected SP → app must use that SP, not the nearest. ❌ (currently broken)
- Veena (L3) confirms: **"Service point selection will be enhanced in the upcoming release (380). Kindly inform the merchant and move the ticket to dev_needed"** (#14).
- Priyanka informs customer of dev release commitment (#15). Ticket moved to `dev_needed` (#16). Customer acknowledges (#17, #18).

## Open Issues

1. **Postnord automation rule "Set Default Service Point" overrides customer-selected service point** — When both (a) the "Set Default Service Point for PostNord" automation rule is active AND (b) the customer has chosen a service point at checkout, the rule wins and assigns the *closest* SP instead of honoring the customer's choice. The rule needs to act only as a fallback for orders without a customer selection — i.e. customer-selected SP should take precedence. Confirmed reproducible on order #33295 (customer chose "Lähiboksi Kauppakeskus Lauttis 1", app generated label with "K-market Kasinonranta"). Workaround offered: bypass the app and generate label on PostNord's website. Blocked: engineering. Severity: `dev_needed` (release 380 targeted). Area: `label-generation`. (Comments #11, #13, #14)

## Resolved Issues

1. **Removing the automation rule broke the no-customer-choice fallback (initial misdiagnosis)** — Following the original L1 advice to delete the "Set Default Service Point" action caused every order without a customer-selected SP to fail label generation. Reverted on 2026-05-08 by re-enabling the rule, restoring the closest-SP fallback. Confirmed this was a side-effect of the advice, not a separate bug — the underlying issue is the rule's priority order (it should defer to customer-selected SP, not override it). (Comments #4, #7, #8, #9, #10)
