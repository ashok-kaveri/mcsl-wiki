## User Story

As an **India-based merchant onboarded to Amazon Shipping through Amazon's official Marketing Associate, who followed every support instruction including recharging ₹600 to my Amazon wallet, processing 58 orders and needing Amazon Shipping to fulfill them**,
I want **clear visibility into whether my account configuration issue is on PluginHive's side or Amazon's side, with proactive escalation to Amazon when the error message ("insufficient credit balance") doesn't match reality (wallet has ₹600), and transparent status updates while PluginHive coordinates with Amazon's internal tech team**,
so that **I don't waste 6 days debugging a wallet balance that was never actually the problem, can generate labels and fulfill my 58 pending orders before customers cancel, and don't lose trust in both PluginHive and Amazon because neither could diagnose that my account was misconfigured on a prepaid model instead of standard billing**.

## Acceptance Criteria (Simple)

- [ ] When Amazon Shipping API returns "insufficient credit balance" error, app checks actual wallet balance via API before showing error to merchant
- [ ] If wallet balance is sufficient but API still rejects: app displays "Account configuration issue detected. Escalating to Amazon Shipping support team. Ticket: #XXXXX"
- [ ] App auto-creates internal escalation ticket to Amazon Shipping partner team with: merchant email, Amazon Shipping account ID, error message, wallet balance screenshot
- [ ] Merchant receives email: "We've identified this is an Amazon account configuration issue (not a wallet balance issue). Our partner team is working directly with Amazon. Expected resolution: 24-48 hours."
- [ ] Support can view merchant's Amazon wallet balance and account configuration type (prepaid vs standard) in app admin panel to speed diagnosis

---

## Acceptance Criteria (Given/When/Then)

### Scenario: API error doesn't match wallet balance reality
- **Given** merchant has ₹600 in Amazon Shipping wallet (sufficient balance)
- **And** merchant attempts to generate Amazon Shipping label
- **When** Amazon API returns "insufficient credit balance" error
- **Then** app queries Amazon wallet balance API to verify actual balance
- **And** displays: "Error received: 'Insufficient balance'. Actual balance: ₹600. This is an account configuration issue, not a payment issue. We're escalating to Amazon."
- **And** auto-creates escalation ticket to Amazon Shipping partner team

### Scenario: Merchant recharged wallet but error persists
- **Given** merchant received "insufficient balance" error and recharged ₹600 as instructed
- **When** merchant retries label generation 1 hour after recharge
- **Then** app verifies new balance (₹600) via API before calling label generation endpoint
- **And** if error persists with sufficient balance: immediately escalates to Amazon (no further merchant action required)

### Scenario: Support diagnoses prepaid vs standard billing configuration
- **Given** support is troubleshooting persistent "insufficient balance" error
- **When** support views merchant account in app admin panel
- **Then** admin panel displays: "Amazon Shipping Account Type: Prepaid Wallet" or "Standard Billing"
- **And** shows current wallet balance and last successful wallet API call timestamp
- **And** provides "Escalate to Amazon Partner Team" button for one-click escalation
