## User Story

As a **high-volume WooCommerce merchant in Quebec generating 10,000+ labels per month who just gained Delivro as a new carrier but can't use it reliably because label generation fails whenever an order is missing a Delivro-required field (pickup instructions, package type) — and because the app applies the same box defaults for every carrier instead of letting me configure packaging per carrier**,
I want **configurable default values for Delivro-required shipment fields that auto-populate when order data is incomplete, and carrier-specific packaging configuration so Delivro shipments use appropriate box dimensions without affecting my CanPar settings**,
so that **my team processes Delivro labels at the same speed as CanPar without manually filling in missing fields on every order, I don't get API validation failures on orders with partial data, and the right packaging dimensions apply automatically based on which carrier I select**.

---

## Acceptance Criteria (Simple)

- [ ] Merchant can configure default/fallback values per carrier for required fields (pickup instructions, reference numbers, package type)
- [ ] Carrier-specific packaging configuration allows different default box dimensions for Delivro vs CanPar
- [ ] Label generation succeeds for orders with incomplete data by merging order fields with configured carrier defaults
- [ ] Default values are carrier-scoped — setting Delivro defaults does not affect CanPar or other carriers
- [ ] ADN Communications (vapewolinak.com) can process Delivro labels in bulk without per-order field editing

---

## Acceptance Criteria (Given/When/Then)

### Scenario: Defaults fill missing Delivro fields
- **Given** a WooCommerce order missing "pickup instructions" required by Delivro's API
- **When** the merchant generates a Delivro label
- **Then** the configured default pickup instructions are used
- **And** the label generates successfully without validation error

### Scenario: Carrier-specific box selection
- **Given** Delivro default box = 30x20x15cm, CanPar default box = 40x30x20cm
- **When** a label is generated with Delivro selected
- **Then** 30x20x15cm dimensions are applied automatically
- **And** switching to CanPar on the same order applies 40x30x20cm

### Scenario: Bulk processing with defaults
- **Given** 50 orders queued for Delivro, 12 missing reference number field
- **When** batch label generation runs
- **Then** all 50 labels generate — 12 use configured default reference number
- **And** batch summary shows "12 orders used default values" for audit
