## User Story

As a **merchant who has already opened one support ticket for label generation failures**, I want **the system to provide clear diagnostic information showing WHAT failed and WHY (not just "label generation failed")**, so that **I can either fix the issue myself or give support the specific error context to resolve it faster without playing phone tag with carriers**.

---

## Acceptance Criteria (Given/When/Then)

### Scenario: Label generation failure includes specific error detail

- **Given** I attempt to generate a shipping label and the operation fails
- **When** the system shows an error message
- **Then** it displays the specific reason (e.g., "invalid tracking number format", "carrier account missing", "address validation failed")
- **And** the error message includes a suggested fix or support ticket link

### Scenario: Retry after carrier issue resolution

- **Given** I've contacted BlueDart and resolved their account issue
- **When** I re-attempt label generation for the same order
- **Then** the label generates successfully
- **And** I don't have to re-open a support ticket or wait for support to test

### Scenario: Prevention: No vague "generation failed" errors

- **Given** any label generation failure
- **When** the error is logged
- **Then** the system never shows generic "label generation failed" to the merchant
- **And** support can see the detailed backend error for debugging
