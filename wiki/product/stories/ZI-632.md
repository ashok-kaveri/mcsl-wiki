## User Story

As a **Premium-plan enterprise retailer shipping ~1,500 labels/month from Saudi Arabia with an existing Aramex setup**,
I want **AJEX available as a configured carrier in MCSL so I can generate labels and manage shipments through their network alongside Aramex**,
so that **my logistics team stops manually creating AJEX shipments outside the app, which delays fulfillment, splits our shipping workflow across two systems, and risks losing the carrier partnership that both AJEX and our IT team invested weeks coordinating**.

## Acceptance Criteria (Simple)

- [ ] AJEX appears as a selectable carrier in the MCSL carrier configuration screen
- [ ] Giordano Saudi Arabia (t301ez-mq.myshopify.com) can enter AJEX API credentials and successfully authenticate
- [ ] Label generation produces valid AJEX shipping labels for both COD and Pre-Paid shipment types
- [ ] Saudi Arabia city/district mapping from AJEX is loaded and used for address validation during label creation
- [ ] AJEX shipment tracking numbers are returned and visible in the order grid after label generation

---

## Acceptance Criteria (Given/When/Then)

### Scenario: AJEX credential configuration and authentication
- **Given** a Shopify MCSL merchant on the carrier configuration screen
- **When** they select AJEX as a carrier and enter valid AJEX API credentials
- **Then** the app authenticates against the AJEX API and displays a success confirmation with the account details

### Scenario: AJEX label generation for COD and Pre-Paid shipments
- **Given** a merchant with configured AJEX credentials and an unfulfilled order shipping to a Saudi Arabia address
- **When** they generate a shipping label selecting AJEX as the carrier with either COD or Pre-Paid payment mode
- **Then** a valid AJEX shipping label is created, a tracking number is returned, and the label PDF is downloadable

### Scenario: City/district validation for Saudi Arabia, Bahrain, and UAE
- **Given** a merchant generating an AJEX label with a destination address in Saudi Arabia, Bahrain, or UAE
- **When** the destination city or district does not match AJEX's mapping files
- **Then** the app displays a clear validation error identifying the mismatched field before the label request is sent to AJEX
