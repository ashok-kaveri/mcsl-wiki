## User Story

As a **new Portuguese Shopify merchant who installed MCSL specifically because Iberomail is my only commercial carrier and I already have their API credentials in hand**,
I want **to connect my Iberomail account inside the app for both domestic and international shipping without waiting months for a bespoke integration**,
so that **I can actually start using the shipping app I signed up for — generating labels and showing rates for 200+ parcels/month — instead of leaving my store without a functional carrier and churning before I've ever printed a label**.

## Acceptance Criteria (Simple)

- [ ] mb-cork.myshopify.com can connect their Iberomail account using their existing API credentials from the Add Carrier page
- [ ] Iberomail rates (domestic Portugal + international) surface at checkout and on the rate-shopping view, priced correctly
- [ ] Labels can be generated for Iberomail shipments (domestic + international formats) with shipment data pushed via Iberomail's API
- [ ] The new-carrier request record for Iberomail tracks: API docs URL, carrier contact, merchant volume, status (scoped/committed/shipped), and expected ETA — visible to both the customer and product team
- [ ] A Portugal-region fallback (e.g., generic API-connector option or paid custom integration quote) is offered within 14 days if the built-in integration is delayed, so the customer doesn't sit idle

## Acceptance Criteria (Given/When/Then)

### Scenario: Deliver Iberomail integration for mb-cork
- **Given** Iberomail API docs and a technical contact have been collected, and the integration has been scoped and built
- **When** the merchant enters their Iberomail API credentials on the Add Carrier page
- **Then** the credentials validate successfully
- **And** the merchant can immediately request rates and generate a label for a Portugal-domestic order

### Scenario: Handle both domestic and international Iberomail flows
- **Given** an Iberomail-connected store has both Portugal-domestic and international (EU / worldwide) orders
- **When** rate shopping runs for each order type
- **Then** the correct Iberomail service codes and rates are returned for each shipment profile
- **And** labels generated for international orders include the customs fields Iberomail requires

### Scenario: Don't strand new carriers' first merchants
- **Given** a new-carrier request has been open for 14 days without being committed to dev
- **When** the system auto-reviews the request
- **Then** the customer is offered either (a) a generic REST-API connector with manual mapping or (b) a paid custom-integration scope — and sees the decision in-app
- **And** the product team receives a weekly digest of long-pending carrier requests
