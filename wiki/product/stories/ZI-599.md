## User Story

As a **shipping app integration partner evaluating PluginHive's platform for PostNord's new Delivery Options API rollout across Nordic e-commerce**,
I want **checkout-time access to PostNord's real-time service availability and estimated delivery windows so merchants using our carrier see accurate, filtered delivery choices**,
so that **Nordic merchants on Shopify stop showing stale or unavailable PostNord options at checkout, which erodes buyer confidence and drives cart abandonment**.

---

## Acceptance Criteria (Simple)

- [ ] PostNord Delivery Options API is called at checkout with origin address, pickup-ready timestamp, and receiver info
- [ ] Checkout displays only PostNord delivery services that are actually available for the given shipment parameters
- [ ] Estimated delivery day and latest delivery time are shown per available PostNord service option
- [ ] Unavailable or expired PostNord services are filtered out and never displayed to the buyer
- [ ] Integration handles API errors or timeouts gracefully — falls back to static PostNord rates without breaking checkout

---

## Acceptance Criteria (Given/When/Then)

### Scenario: Real-time PostNord delivery options displayed at checkout
- **Given** a Shopify merchant has PostNord configured as a carrier in MCSL and a buyer reaches checkout with a Finnish delivery address
- **When** the checkout requests shipping rates
- **Then** MCSL calls the PostNord Delivery Options API with origin address, pickup-ready timestamp, and receiver info, and displays only available PostNord services with estimated delivery day and latest delivery time

### Scenario: Unavailable services filtered from checkout
- **Given** the PostNord DO API returns a mix of available and unavailable delivery services for a given origin-destination pair
- **When** the buyer views shipping options at checkout
- **Then** only services marked as available by the DO API are shown; unavailable services are excluded entirely

### Scenario: API failure does not break checkout
- **Given** a Shopify merchant has PostNord configured and the PostNord Delivery Options API is unreachable or returns an error
- **When** a buyer reaches checkout
- **Then** MCSL falls back to existing static PostNord rate configuration and logs the API failure for monitoring, without delaying or breaking the checkout experience
