## User Story

As a **WooCommerce merchant shipping via Delivro in Quebec where delivery drivers rely on printed route codes to navigate efficient delivery sequences**,
I want **every Delivro label generated through the plugin to consistently include the route code that the Dispatch Science API assigns to the shipment**,
so that **my drivers do not have to cross-reference Delivro's portal to manually look up route codes for labels that are missing them, which slows down dispatch and causes delivery delays**.

---

## Acceptance Criteria (Simple)

- [ ] Every Delivro label generated through the WSS plugin includes the route code field when the Dispatch Science API response contains a route code value
- [ ] When the Dispatch Science API returns a shipment response without a route code (e.g., for certain service types or destinations), the label renders cleanly without a blank or malformed route code field
- [ ] The route code is positioned consistently on the label in the same location as other Delivro-printed labels (matching the Delivro portal's label format)
- [ ] Batch-generated Delivro labels across multiple orders all include their respective route codes without any labels being skipped or rendered without the code
- [ ] A diagnostic log entry records the route code value (or its absence) for each Delivro label generation request to support troubleshooting inconsistencies

---

## Acceptance Criteria (Given/When/Then)

### Scenario: Label includes route code from API response
- **Given** a WooCommerce order assigned to the Delivro carrier, and the Dispatch Science API returns a shipment response containing a route code value (e.g., "QC-MTL-07-A")
- **When** the plugin generates and renders the shipping label PDF for this order
- **Then** the route code "QC-MTL-07-A" appears on the printed label in the designated route code field, matching the position and format used by Delivro's own portal labels

### Scenario: Label handles missing route code gracefully
- **Given** a WooCommerce order assigned to the Delivro carrier, and the Dispatch Science API returns a shipment response where the route code field is null or absent
- **When** the plugin generates the shipping label for this order
- **Then** the label renders without a route code field (no blank space, no placeholder text), and the plugin logs a warning noting the route code was not provided by the API for this shipment

### Scenario: Batch generation produces consistent route codes across all labels
- **Given** a batch of 20 WooCommerce orders selected for Delivro label generation, where the Dispatch Science API returns route codes for 18 of the 20 shipments
- **When** the merchant generates labels for all 20 orders in a single batch operation
- **Then** all 18 labels with route codes display them correctly, the 2 labels without route codes render cleanly without the field, and the batch summary indicates which orders had route codes and which did not
