## User Story

As a **high-volume Indian D2C merchant on the $49 Premium plan shipping ~2,800 labels/month through BlueDart and Delhivery**,
I want **same-day and 30–60 minute last-mile delivery via BlitzNow in metro cities alongside my existing carriers**,
so that **I can compete with marketplace delivery speeds, stop losing metro customers to competitors who offer instant delivery, and justify the 12-month commitment I made to get this built**.

## Acceptance Criteria (Simple)

- [ ] BlitzNow appears as a selectable carrier in the MCSL carrier configuration panel
- [ ] Merchant can authenticate with BlitzNow using API credentials (key + secret) and the connection persists across sessions
- [ ] Rate shopping returns BlitzNow rates alongside existing carriers (BlueDart, Delhivery) for eligible Indian metro pin codes
- [ ] Label generation produces a valid BlitzNow shipping label with trackable AWB number
- [ ] Shipment tracking returns real-time status updates from BlitzNow's tracking API
- [ ] Belvish (belvish.myshopify.com) can generate a BlitzNow label for a test order end-to-end before public rollout
- [ ] Integration handles BlitzNow API errors gracefully — credential failures, service-area mismatches, and rate timeouts surface clear user-facing messages instead of silent failures

---

## Acceptance Criteria (Given/When/Then)

### Scenario: Merchant connects BlitzNow credentials
- **Given** a Shopify MCSL merchant navigates to carrier configuration
- **When** they select BlitzNow and enter valid API key, secret, and account ID
- **Then** the app validates the credentials against BlitzNow's auth endpoint, shows a success confirmation, and persists the connection

### Scenario: Rate shopping includes BlitzNow for eligible orders
- **Given** a merchant has BlitzNow connected and an order ships to a BlitzNow-serviceable metro pin code
- **When** the merchant triggers rate shopping for that order
- **Then** BlitzNow rates appear alongside BlueDart and Delhivery rates with estimated delivery time and cost

### Scenario: Label generation and tracking for BlitzNow shipment
- **Given** a merchant selects a BlitzNow rate for an order
- **When** they generate the shipping label
- **Then** a valid label with AWB number is produced, the order is marked as fulfilled in Shopify, and tracking events from BlitzNow's API are available in the tracking panel

### Scenario: Non-serviceable pin code handling
- **Given** an order ships to a pin code outside BlitzNow's service area
- **When** the merchant triggers rate shopping for that order from the order grid or order summary
- **Then** BlitzNow is excluded from the results with a clear reason ("Pin code not serviceable by BlitzNow") and other carriers still return rates normally

### Scenario: API credential failure
- **Given** a merchant enters invalid or expired BlitzNow credentials
- **When** they attempt to save the carrier connection
- **Then** the app displays a specific error message ("Authentication failed — verify your BlitzNow API key and secret") without saving invalid credentials
