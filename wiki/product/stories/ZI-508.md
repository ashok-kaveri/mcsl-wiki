## Open Issues

1. **Edit Package feature unavailable for FedEx REST API accounts** — After migrating to FedEx REST, customers don't see the "Edit Package" UI that was available on the legacy FedEx SOAP integration. Functional regression for FedEx REST-migrated stores; affects ability to adjust package dimensions/weight on a per-order basis before label generation. Blocked: engineering (UI flow needs to be ported to the REST account model). Severity: medium (workflow regression affecting all FedEx REST migrations, of which this is the leading edge per the April 2026 deadline). Area: carrier-migration. (Comments #20, #22, #37)

2. **Plugin passes customs/declared value on FedEx REST requests for domestic shipments without insurance enabled** — In the original report, FedEx 2 Day quoted $16.88 in plugin vs $11.93 on FedEx.com for a domestic order with no insurance enabled in plugin settings and no customs value on products. Root cause: plugin's FedEx REST request included a declared value, which FedEx interpreted as a request for higher-limit-of-liability coverage and added a $4.95 insurance surcharge. Veena's clarification: FedEx applies the surcharge automatically only for shipments over $500. Engineering reported a fix on 2026-04-02; customer was asked to re-test on 2026-04-03. No customer confirmation in subsequent comments — needs verification this is closed for Duong's account or whether the WSS release fully shipped. Blocked: customer verification / release tracking. Severity: high (silent over-charging across any FedEx REST customer with domestic shipments over $500). Area: rate-shopping. (Comments #15, #20, #28, #29, #31, #34)
