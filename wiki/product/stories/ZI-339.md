## User Story

As a **high-volume merchant shipping bulk identical items that require multi-box packing (vinyl records, books, apparel cases) on a $49/month plan generating 292 labels in April alone**,
I want **to define reusable box templates with capacity limits ("Box A holds 20 vinyl records, Box B holds 50 t-shirts") so the app automatically calculates how many boxes an order requires and applies correct dimensions per box**,
so that **I don't waste 10+ minutes per order manually entering box dimensions for 30-item shipments, avoid carrier billing disputes from incorrect package sizes, and can fulfill high-volume orders at the speed my business requires without abandoning the app for manual carrier portals that support box templates**.

---

## Acceptance Criteria (Simple)

- [ ] Merchant can create custom box templates with name, dimensions (L×W×H), weight capacity, and item capacity
- [ ] During label generation for multi-item orders, app suggests box template(s) based on total items/weight
- [ ] If order exceeds single box capacity, app automatically calculates number of boxes needed and splits shipment accordingly
- [ ] Box template selection persists for product/SKU combinations (e.g., "Vinyl orders always use Box Type A")
- [ ] Generated labels show correct per-box dimensions and box count (e.g., "Package 1 of 2: 40×40×20cm")

---

## Acceptance Criteria (Given/When/Then)

### Scenario: Merchant creates box template for bulk vinyl orders
- **Given** merchant ships vinyl records (31.5cm × 31.5cm each) in custom boxes holding 20 units
- **When** merchant navigates to Settings → Packaging → Add Box Template
- **Then** merchant can enter box name "Vinyl Box 20", dimensions (40×40×20cm), max weight (10kg), max items (20 units)
- **And** template is saved and available for label generation

### Scenario: Order exceeds single box capacity, app auto-calculates multi-box shipment
- **Given** merchant has "Vinyl Box 20" template configured (capacity: 20 records)
- **And** order contains 35 vinyl records
- **When** merchant generates shipping label
- **Then** app displays: "This order requires 2 boxes: Box 1 (20 items), Box 2 (15 items)"
- **And** generates 2 labels with correct dimensions per box, marked "Package 1 of 2" and "Package 2 of 2"

### Scenario: Merchant assigns box template to specific product/SKU
- **Given** merchant has multiple box templates configured
- **When** merchant edits product/SKU shipping settings
- **Then** merchant can select default box template for that product (e.g., "Always use Vinyl Box 20 for SKU: VINYL-001")
- **And** future orders for that product auto-select the assigned box template during label generation
