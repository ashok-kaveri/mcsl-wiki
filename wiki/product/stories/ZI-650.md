## User Story

As a **high-volume WooCommerce merchant processing 100-150 orders daily who needs to filter orders by date, status, or carrier to find specific shipments quickly**,
I want **the order filter panel in the WSS plugin to remain fully visible and functional when I scroll through my order list**,
so that **I can apply and adjust filters without the filter controls being clipped or hidden behind other UI elements, which currently forces me to scroll back to the top of the page every time I want to change a filter**.

---

## Acceptance Criteria (Simple)

- [ ] The order filter panel remains fully visible and accessible when the user scrolls down through a long list of orders in the WSS plugin
- [ ] Filter dropdowns, date pickers, and action buttons within the filter panel do not get clipped, hidden, or rendered behind other page elements at any scroll position
- [ ] The fix works consistently across Chrome, Firefox, and Safari on desktop viewports (1280px and above)
- [ ] The filter panel behavior does not interfere with the WooCommerce admin sidebar, admin bar, or other WordPress admin UI elements
- [ ] The filter panel remains usable on screens with 768px height (common laptop viewport) without requiring the user to scroll within the filter panel itself

---

## Acceptance Criteria (Given/When/Then)

### Scenario: Filter panel stays visible while scrolling through orders
- **Given** a WooCommerce merchant is viewing the WSS plugin's order list page with 50+ orders loaded and the filter panel is visible at the top of the order grid
- **When** the merchant scrolls down past the first 20 orders so the filter panel would normally scroll out of the viewport
- **Then** the filter panel remains accessible (either by staying fixed/sticky or by providing a clearly visible mechanism to reopen it) and all filter controls are fully interactive without visual clipping

### Scenario: Filter dropdowns render completely without being cut off
- **Given** the merchant has scrolled to a position where the filter panel is at the top edge of the viewport and clicks on a filter dropdown (e.g., "Carrier" or "Status")
- **When** the dropdown menu expands to show its options
- **Then** the full dropdown is visible within the viewport, not clipped by overflow:hidden, z-index stacking, or container boundaries, and the merchant can select any option without the dropdown disappearing or being obscured

### Scenario: Applying a filter after scrolling refreshes the order list correctly
- **Given** the merchant has scrolled down the order list and then uses the filter panel to apply a date range filter (e.g., "Last 7 days")
- **When** the filter is applied and the order list refreshes with the filtered results
- **Then** the page scrolls back to the top of the filtered results, the filter panel shows the active filter state, and the order count reflects the filtered subset
