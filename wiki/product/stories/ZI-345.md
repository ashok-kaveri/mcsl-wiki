## User Story

As a **Hong Kong merchant generating 202 labels in April, already frustrated by a CSV import bug that wiped my product data, now completely blocked from accessing the app on the morning of April 27 when I need to fulfill orders**,
I want **automatic app health monitoring that detects infinite loading states within 60 seconds, shows a helpful error message ("The app isn't loading. Refreshing the page may help. If the issue persists, contact support with error code: LOAD-TIMEOUT-385311"), and auto-notifies support with browser console logs so they can diagnose without asking me for screenshots**,
so that **I don't lose an entire morning of fulfillment time staring at a loading spinner, know immediately if it's my browser or the app, and can try basic troubleshooting (refresh, clear cache, different browser) before opening a ticket that support can't diagnose without more information**.

---

## Acceptance Criteria (Simple)

- [ ] App detects infinite loading state after 60 seconds (no UI render)
- [ ] After 60 seconds: displays error message with troubleshooting steps and error code
- [ ] Error message includes: "Try refreshing the page. If that doesn't work, clear your browser cache. Still stuck? Contact support with error code: LOAD-{timestamp}"
- [ ] App auto-submits crash report to support with: merchant email, store URL, browser/OS, console error logs, last successful API call
- [ ] Support dashboard shows "App Loading Failures" metric with affected stores in real-time

---

## Acceptance Criteria (Given/When/Then)

### Scenario: App fails to load within 60 seconds
- **Given** merchant navigates to Multi Carrier app from Shopify admin
- **And** app's JavaScript fails to initialize (infinite spinner)
- **When** 60 seconds pass without UI rendering
- **Then** loading spinner is replaced with error message: "The app isn't loading. This could be a browser issue or a temporary service problem."
- **And** shows troubleshooting steps: "1. Refresh this page. 2. Clear browser cache. 3. Try a different browser. 4. Contact support if issue persists: Error code LOAD-{timestamp}"

### Scenario: App auto-reports loading failure to support
- **Given** app detected infinite loading state
- **When** error message is displayed to merchant
- **Then** app automatically sends crash report to support with: store URL, merchant email, browser/OS/device, JavaScript console errors, last API response status
- **And** support dashboard shows real-time alert: "Store fa8d9f.myshopify.com - app loading failure detected (LOAD-{timestamp})"

### Scenario: Merchant successfully troubleshoots loading issue
- **Given** merchant sees loading timeout error message
- **When** merchant clicks "Refresh Page" or clears cache as instructed
- **Then** app loads successfully within 10 seconds
- **And** app logs successful recovery to analytics (to track if loading failures are transient or persistent)
