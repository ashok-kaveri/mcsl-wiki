## User Story

As a **Shopify merchant managing hundreds of products with varied packaging dimensions across multiple categories**,
I want **to select multiple products on the Products page and assign individual dimensions to each in a single bulk-edit session**,
so that **I can accurately configure shipping dimensions for my entire catalogue without opening and editing each product one at a time, saving hours of repetitive manual work each week**.

---

## Acceptance Criteria (Simple)

- [ ] Merchant can select multiple products/variants on the Products page regardless of whether they share the same current dimensions
- [ ] A bulk-edit modal or inline editor allows entering distinct length, width, height, and weight values for each selected product independently
- [ ] Saving the bulk edit persists all per-product dimension changes in a single operation without requiring page reload
- [ ] Products with existing dimensions retain their values if the merchant does not modify them during the bulk edit
- [ ] The bulk-edit workflow handles variant-level dimensions when a product has multiple shipping-relevant variants

---

## Acceptance Criteria (Given/When/Then)

### Scenario: Bulk edit with distinct dimensions per product
- **Given** a merchant has selected 5 products on the Products page, each with different current dimensions
- **When** they click the bulk-edit action and enter unique length, width, height, and weight values for each of the 5 products
- **Then** all 5 products are updated with their respective individual dimensions in a single save operation

### Scenario: Partial update leaves unmodified products unchanged
- **Given** a merchant has selected 10 products and modifies dimensions for only 3 of them in the bulk editor
- **When** they save the bulk edit
- **Then** only the 3 modified products are updated, and the remaining 7 retain their original dimension values

### Scenario: Variant-level dimension assignment in bulk edit
- **Given** a merchant selects a product that has 3 variants (e.g. Small, Medium, Large) each requiring different box sizes
- **When** they expand the product in the bulk editor and assign distinct dimensions to each variant
- **Then** each variant stores its own shipping dimensions independently and those dimensions are used during rate calculation and label generation
