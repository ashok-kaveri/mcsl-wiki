## User Story

As a **Swedish merchant generating 332 labels in April (averaging 10+ orders per day), relying on the "open orders" folder to know what still needs fulfillment**,
I want **fulfilled orders to automatically move out of the open orders folder within 60 seconds of fulfillment, with a manual "Refresh Order Status" button as a failsafe if sync is delayed**,
so that **I don't waste 15 minutes every morning cross-referencing the open folder against Shopify's fulfilled list to figure out what actually needs labels, accidentally reprint labels for already-fulfilled orders (wasting $3-5 per duplicate label), or lose trust in the app's order organization and resort to manual spreadsheet tracking which defeats the purpose of paying $29/month for automation**.

---

## Acceptance Criteria (Simple)

- [ ] When order status changes to "Fulfilled" in Shopify, app removes it from open orders folder within 60 seconds
- [ ] App listens to Shopify webhook: "orders/fulfilled" to trigger immediate folder update (not polling)
- [ ] If webhook is missed/delayed: fallback polling every 5 minutes syncs order statuses
- [ ] Open orders folder shows "Last synced: 2 minutes ago" timestamp with manual "Refresh Now" button
- [ ] If manual refresh is clicked and orders move: app logs the incident for engineering to investigate webhook delivery failures

---

## Acceptance Criteria (Given/When/Then)

### Scenario: Order is fulfilled, moves out of open folder automatically
- **Given** order #1007985711 is in the "open orders" folder
- **And** merchant generates shipping label and marks order as fulfilled in Shopify
- **When** Shopify sends "orders/fulfilled" webhook to app
- **Then** within 60 seconds, order #1007985711 is removed from open orders folder
- **And** moved to "fulfilled orders" folder (or removed from view if no fulfilled folder exists)

### Scenario: Webhook delivery failure, fallback polling syncs order
- **Given** Shopify webhook for "orders/fulfilled" fails to deliver to app
- **And** order #1007985711 is fulfilled but still showing in open orders
- **When** app's fallback polling runs (every 5 minutes)
- **Then** polling detects order #1007985711 status changed to fulfilled
- **And** removes it from open orders folder
- **And** logs webhook delivery failure for investigation

### Scenario: Merchant manually refreshes order status
- **Given** merchant notices order #1007985711 fulfilled on April 20 still in open folder on April 22
- **When** merchant clicks "Refresh Order Status" button in open orders folder
- **Then** app immediately queries Shopify for latest order statuses (not waiting for next polling interval)
- **And** moves fulfilled orders out of open folder within 5 seconds
- **And** displays success message: "3 fulfilled orders moved. If this happens often, contact support."
