## Acceptance Criteria (Detailed — Given/When/Then)

### Scenario 1: Previously failing orders now generate successfully
- **Given** the merchant has orders that were failing label generation before the fix
- **When** the merchant selects any of those orders and clicks "Generate Label"
- **Then** the label generates successfully on the first attempt
- **And** the same failure pattern from ticket #379648 does not recur

### Scenario 2: Batch generation doesn't silently fail on some orders
- **Given** the merchant selects 10 orders for batch label generation
- **When** the batch completes
- **Then** each order either has a label with tracking number, or a specific per-order error explaining why it failed (not a blanket "generation failed" for the whole batch)

### Scenario 3: Failure gives the merchant enough information to self-resolve
- **Given** a label fails because of a carrier-side rejection (e.g., unsupported destination, weight limit)
- **When** the error is shown to the merchant
- **Then** the message includes: the carrier's rejection reason, which field to correct, and a "Retry" action — not just an error code
