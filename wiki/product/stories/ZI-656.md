## User Story

As a **Shopify merchant using multiple fulfillment systems where orders sometimes get marked as fulfilled by an external system before I can generate shipping labels in MCSL**,
I want **a self-service action to reset orders from "Externally Fulfilled" back to "Processing" status directly in the MCSL app**,
so that **I can generate shipping labels for these orders without waiting for support to manually reset each one, which delays my shipments and doesn't scale as my order volume grows**.

---

## Acceptance Criteria (Simple)

- [ ] A "Reset to Processing" action is available on orders with "Externally Fulfilled" status in the MCSL order grid or order detail view
- [ ] After reset, the order appears in the normal Processing queue and is eligible for label generation, rate shopping, and all standard order actions
- [ ] The reset action is available as both a single-order action and a bulk action for multiple externally fulfilled orders
- [ ] The Shopify fulfillment status is updated accordingly (unfulfilled) so that MCSL and Shopify remain in sync
- [ ] An audit trail entry is recorded when an order is reset from Externally Fulfilled, capturing who reset it and when

---

## Acceptance Criteria (Given/When/Then)

### Scenario: Merchant resets an externally fulfilled order to Processing
- **Given** an order was fulfilled by an external system and appears as "Externally Fulfilled" in the MCSL order grid
- **When** the merchant selects the order and chooses "Reset to Processing" from the actions menu
- **Then** the order status changes to "Processing" in MCSL, the Shopify fulfillment is reverted to unfulfilled, and the order becomes available for label generation

### Scenario: Bulk reset of multiple externally fulfilled orders
- **Given** five orders were all marked as "Externally Fulfilled" by an external system that incorrectly triggered Shopify fulfillment
- **When** the merchant selects all five orders in the grid and applies the "Reset to Processing" bulk action
- **Then** all five orders transition to "Processing" status and appear in the standard fulfillment queue, ready for label generation

### Scenario: Reset does not affect orders fulfilled through MCSL
- **Given** an order was properly fulfilled through MCSL with a generated shipping label and tracking number
- **When** the merchant views the order actions menu
- **Then** the "Reset to Processing" action is not available for this order, preventing accidental reversal of a legitimately fulfilled shipment
