## Acceptance Criteria (Given/When/Then)

### Scenario: PostNord international label and CN22 print on single page
- **Given** lobloo.com (customer #379784) generates a PostNord international label to any country requiring CN22 customs form
- **When** the label PDF is generated and downloaded
- **Then** the shipping label and CN22 customs document appear on a single PDF page
- **And** both documents are properly formatted and readable when printed
- **And** the customer can print the combined document without manual merging

### Scenario: All PostNord international shipments use single-page format
- **Given** any Shopify store using the app generates PostNord international labels
- **When** the destination country requires customs documentation (CN22 or CN23)
- **Then** the label and customs form are combined in a single PDF request to PostNord API
- **And** the API returns the combined format as specified in PostNord documentation
- **And** the merchant receives one printable page per international shipment

### Scenario: API integration standardization prevents future format issues
- **Given** PostNord API documentation specifies the combined label+customs format parameters
- **When** the plugin requests label generation for international shipments
- **Then** the correct API parameters are sent to request combined format (not separate documents)
- **And** automated tests validate the PDF structure matches PostNord's single-page specification
- **And** if PostNord changes the API format, integration tests catch the mismatch before customer impact
