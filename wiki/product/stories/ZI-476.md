## User Story

As a **merchant processing multiple shipments per day who relies on accurate manifest data for USPS end-of-day processing**,
I want **the USPS manifest to show the correct count of orders and include all fulfilled shipments without missing any**,
so that **my end-of-day reconciliation matches actual pickups, I don't face discrepancies with USPS billing, and my warehouse team can trust the manifest count for quality control**.

---

## Acceptance Criteria (Simple)

- [ ] USPS manifest PDF displays accurate shipment count matching the actual number of labels included (e.g., 15 shipments shows "15" not a different number)
- [ ] All tracking numbers in the manifest batch appear in the manifest PDF without duplicates or omissions
- [ ] Customer's WooCommerce store (#368959) can generate manifests with correct order counts for any batch size
- [ ] Pre-generation validation checks expected vs actual shipment count and warns merchant of any discrepancies
- [ ] Manifest count accuracy applies across all carriers (USPS, FedEx, UPS) and all batch sizes (1-1000+ shipments)

---

## Acceptance Criteria (Given/When/Then)

### Scenario: USPS manifest displays accurate order count matching actual shipments
- **Given** a merchant generates a USPS end-of-day manifest with 15 shipments
- **When** they download and open the manifest PDF
- **Then** the document header shows "15 shipments" (or equivalent correct count)
- **And** the manifest lists all 15 tracking numbers without duplicates or omissions
- **And** no orders are missing from the manifest that were included in the batch

### Scenario: Manifest count accuracy across all carriers and batch sizes
- **Given** any merchant creates a manifest for any carrier (USPS, FedEx, UPS) with any number of shipments (1-1000+)
- **When** the manifest is generated and downloaded
- **Then** the order count in the manifest matches the actual number of labels included
- **And** the count remains accurate whether manifests are generated once daily or multiple times per day
- **And** partial manifests (subset of orders) show correct subset counts

### Scenario: Automated validation prevents manifest count discrepancies
- **Given** the manifest generation process
- **When** a manifest is created
- **Then** the system performs pre-generation count validation (expected vs actual)
- **And** any mismatch triggers a warning before manifest finalization
- **And** merchants can review the discrepancy and correct it before submitting to the carrier
