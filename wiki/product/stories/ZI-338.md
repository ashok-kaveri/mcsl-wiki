## User Story

As a **merchant shipping internationally with contractual duty payment terms (DDP/DDU) built into my FedEx account**,
I want **every carrier configuration setting preserved during API migration, or to receive explicit warnings about which settings will reset and require re-validation**,
so that **I don't unknowingly violate my shipping contracts, incur unexpected duty charges that I'm contractually prohibited from billing customers for, and have to choose between eating the cost or damaging customer relationships by chasing payment after delivery**.

---

## Acceptance Criteria (Simple)

- [ ] FedEx SOAP → REST migration preserves duty payment configuration (Sender/Recipient/Third Party) without reset
- [ ] If settings cannot be preserved: migration wizard displays red-banner warning listing ALL settings that will reset, forcing acknowledgment before proceeding
- [ ] Post-migration confirmation screen shows before/after comparison of critical billing settings (duty payor, freight payor, billing account)
- [ ] Migration documentation explicitly lists duty payment config in "Settings That Will Reset" section with screenshot
- [ ] Merchant receives email after migration with checklist: "Verify these 5 settings before generating your first label"

---

## Acceptance Criteria (Given/When/Then)

### Scenario: Merchant migrates FedEx account from SOAP to REST
- **Given** merchant has FedEx configured with "Recipient" duty payment (DDU terms)
- **And** merchant initiates FedEx REST API migration via app settings
- **When** migration completes successfully
- **Then** duty payment configuration remains "Recipient" without manual re-entry
- **And** merchant can generate first post-migration label with correct duty billing

### Scenario: Migration cannot preserve settings (API incompatibility)
- **Given** certain FedEx settings have no equivalent in REST API
- **When** merchant reaches migration step where settings will be lost
- **Then** wizard displays prominent warning: "These 3 settings will reset: [Duty Payor Type, X, Y]. You must reconfigure after migration."
- **And** merchant must check "I understand I need to reconfigure these settings" before proceeding
- **And** post-migration screen shows red alerts for each setting requiring attention

### Scenario: Merchant completes migration without re-validating settings
- **Given** merchant migrated FedEx to REST and duty payor was reset to default "Sender"
- **And** merchant generates labels without checking settings
- **When** first label is generated
- **Then** app displays one-time post-migration notice: "FedEx migration complete. Have you verified your duty payment settings? [Check Now] [Dismiss]"
- **And** merchant can click "Check Now" to jump directly to duty payor config with current value highlighted
