## User Story

As a **merchant who has a contract with PostNL for specific services**,
I want **every service in my PostNL contract to be available in the app without waiting for a dev enhancement**,
so that **I can use the cheapest tracked option for letterbox packages instead of overpaying for full parcel services or abandoning the app to generate labels manually on PostNL's portal**.

---

## Acceptance Criteria (Simple)

- [ ] PostNL service 6942 (Boxable Track & Trace) is available for international shipments
- [ ] The app's PostNL service catalog reflects all services available in PostNL's current API — not a hardcoded subset
- [ ] Labels with service 6942 are accepted by PostNL and produce valid tracking numbers
- [ ] This merchant (scandinavianspeciosa.se) can generate labels for NL→SE letterbox shipments using 6942
- [ ] Automation rules can target service 6942 (e.g., "if weight < 2kg and destination = SE, use Boxable T&T")

---

## Acceptance Criteria (Detailed — Given/When/Then)

### Scenario 1: Contracted service appears without manual intervention
- **Given** a merchant's PostNL account supports service code 6942
- **When** the merchant selects PostNL as carrier for an international order
- **Then** "Boxable Track & Trace (6942)" appears alongside other available services
- **And** the merchant doesn't need to contact support to "unlock" it

### Scenario 2: Letterbox shipment is cheaper than full parcel
- **Given** the merchant has a small, lightweight order (< 2kg) going to Sweden
- **When** the merchant views available PostNL services for that order
- **Then** service 6942 is offered at a lower rate than full parcel services (e.g., 4940, 4952)
- **And** the merchant can select it and generate a label that PostNL accepts

### Scenario 3: Future PostNL services don't require dev work
- **Given** PostNL adds a new service code to their API (e.g., a new tracked economy option)
- **When** the merchant's PostNL account is provisioned for that service
- **Then** the new service appears in the app's service list automatically (or via a "refresh services" action)
- **And** no PluginHive dev enhancement or ticket is required
