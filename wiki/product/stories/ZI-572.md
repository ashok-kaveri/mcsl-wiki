## User Story

As a **WooCommerce merchant who is already manually importing orders every few hours as a workaround for broken auto-import, and who has to click the import button multiple times because it doesn't reliably trigger on the first attempt**,
I want **the manual import button to execute a complete order sync on a single click with clear feedback showing how many orders were imported**,
so that **the workaround my team relies on actually works reliably — one click, all pending orders imported, confirmation shown — instead of adding click-frustration on top of the already painful manual process**.

---

## Acceptance Criteria (Simple)

- [ ] Single click on "Import Orders" triggers a complete sync of all pending WooCommerce orders
- [ ] Button shows loading state during import and success message with count: "Imported 12 orders"
- [ ] If import fails, error message explains why (not a silent failure requiring another click)
- [ ] Import completes within 10 seconds for batches of up to 100 orders
- [ ] Jason Ballantyne confirms single-click import works reliably as interim workaround

---

## Acceptance Criteria (Given/When/Then)

### Scenario: Single-click import succeeds
- **Given** 15 new WooCommerce orders pending import
- **When** the merchant clicks "Import Orders" once
- **Then** all 15 orders are imported and visible in the order list
- **And** a success toast shows "15 orders imported successfully"

### Scenario: Import button shows clear progress
- **Given** the merchant clicks "Import Orders"
- **When** the import is processing
- **Then** the button shows a spinner with "Importing..." text (not clickable again)
- **And** the button returns to normal state only after import completes or fails

### Scenario: Failed import shows actionable error
- **Given** the import fails due to API timeout
- **When** the merchant sees the error
- **Then** the message says "Import failed: API timeout. Try again or check WooCommerce API status."
- **And** the merchant can retry with a single click
