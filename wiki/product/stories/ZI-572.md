## User Story

As a **WooCommerce fulfillment team lead managing 87+ shipments/month across time zones with a remote warehouse operator who relies on the plugin to queue today's shipments**,
I want **the manual order import to execute reliably on a single click with clear confirmation of what was pulled in**,
so that **my warehouse team stops wasting 15-20 minutes every 4-6 hours refreshing and re-clicking the import button while orders pile up unshipped and customers wait an extra day for their pet food**.

---

## Acceptance Criteria (Simple)

- [ ] Single click on "Import Orders" triggers a complete sync of all pending WooCommerce orders without requiring page refresh or repeated clicks
- [ ] Button displays a loading/spinner state during import and a success toast showing the count of imported orders (e.g., "Imported 12 orders")
- [ ] If import fails (API timeout, permission error, data access failure), an actionable error message is displayed instead of silently doing nothing
- [ ] Import completes within 10 seconds for batches of up to 100 orders on stores with active webhooks and passing API diagnostics
- [ ] American Natural Premium store (americannaturalpremium.com) confirmed receiving single-click imports reliably for 48 consecutive hours after deployment

---

## Acceptance Criteria (Given/When/Then)

### Scenario: Single-click import pulls all pending orders
- **Given** 15 new WooCommerce orders with "Processing" status exist that have not yet been imported into the WSS plugin
- **When** the merchant clicks the "Import Orders" button once
- **Then** all 15 orders appear in the plugin order list within 10 seconds, and a success toast displays "15 orders imported successfully"

### Scenario: Import button prevents double-click and shows progress
- **Given** the merchant clicks "Import Orders" while a batch of orders is being fetched
- **When** the import is in progress
- **Then** the button enters a disabled/spinner state with "Importing..." text, preventing duplicate requests, and returns to its normal clickable state only after the import completes or fails

### Scenario: Failed import surfaces an actionable error instead of silent failure
- **Given** the WooCommerce REST API is unreachable or the Website Data Access Test returns empty results for queried order IDs
- **When** the merchant clicks "Import Orders"
- **Then** an error message appears within 15 seconds stating the specific failure reason (e.g., "Import failed: unable to retrieve order data from WooCommerce API. Check REST API permissions and connectivity.") and the merchant can retry with a single click
