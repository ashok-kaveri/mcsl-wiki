## User Story

As an **Indonesia-based merchant paying $29/month for shipping automation who ships from Vietnam using UPS**,
I want **the app to capture State/Province for Vietnam ShipFrom addresses so my labels can be generated even when UPS changes their API validation requirements**,
so that **I don't lose all ability to ship my 329 orders when a carrier tightens their API validation overnight, forcing me to either manually create every label in UPS WorldShip or stop fulfilling orders entirely while engineering works on a fix that should have been prevented by robust address form design**.

## Acceptance Criteria (Given/When/Then)

### Scenario: Indonesia merchant generates UPS labels from Vietnam ShipFrom address
- **Given** customer has a UPS-enabled Shopify store shipping from Vietnam locations
- **When** customer creates a label with ShipFrom address in Vietnam including State/Province
- **Then** UPS API accepts the request and returns a valid shipping label
- **And** the label is generated for the customer's specific orders (8676 and others) that previously failed

### Scenario: Merchant in any country affected by carrier API validation changes
- **Given** a carrier (UPS, FedEx, DHL, etc.) changes their API to require a previously-optional address field for a specific country
- **When** merchant attempts to generate a label for that country
- **Then** the app's address form already captures all fields that carrier APIs commonly validate
- **And** if a new required field is introduced, the error message clearly states which field is missing with actionable guidance

### Scenario: Prevention of future carrier API validation failures
- **Given** the app stores ShipFrom addresses for international locations
- **When** displaying address forms for any country
- **Then** all address fields that any integrated carrier might validate are captured (address lines, city, postal code, state/province, country)
- **And** carrier API validation errors are logged with full request/response XML for L3 analysis
- **And** merchants can self-serve add missing address fields without opening support tickets
