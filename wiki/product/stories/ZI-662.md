## User Story

As a **WooCommerce marketplace operator using Dokan to manage multiple vendors across different countries, each with their own DHL Express shipping accounts**,
I want **the PluginHive Shipping (PSS) settings page on the Dokan vendor dashboard to load reliably without configuration errors after the initial multi-vendor setup is complete**,
so that **my vendors can independently manage their shipping settings, view rates, and generate labels from their own dashboard without requiring admin intervention or FTP-level fixes each time the page breaks**.

---

## Acceptance Criteria (Simple)

- [ ] The Dokan vendor dashboard Shipping (PSS) page loads without displaying "PluginHive settings haven't been configured yet" when the plugin has been properly installed and vendor sync completed
- [ ] Each vendor sees only their own carrier accounts and shipping settings on their Dokan dashboard PSS page
- [ ] Vendor-specific origin addresses are used for rate calculation when a customer checks out with items from that vendor
- [ ] The PSS page initialisation handles the Dokan Free plugin variant without requiring manual file-level patches on the server
- [ ] After admin completes the initial multi-vendor setup and carrier registration, no additional FTP or server-side intervention is needed for the vendor dashboard to function

---

## Acceptance Criteria (Given/When/Then)

### Scenario: Vendor dashboard PSS page loads after fresh multi-vendor setup
- **Given** an admin has installed the PluginHive shipping plugin, synced vendors via Dokan, and registered a DHL Express account for the marketplace
- **When** a vendor logs into their Dokan dashboard and navigates to the Shipping (PSS) settings page
- **Then** the page loads successfully showing the vendor's assigned carrier accounts and shipping configuration options without any "not configured" error

### Scenario: Vendor-isolated shipping settings on multi-vendor marketplace
- **Given** a marketplace has three vendors (Vendor A in Portugal, Vendor B in Spain, Vendor C in France) each assigned their country-specific DHL Express account
- **When** Vendor B logs into their Dokan dashboard and views the Shipping (PSS) page
- **Then** Vendor B sees only the Spain DHL Express account configuration and cannot view or modify Vendor A's or Vendor C's carrier settings

### Scenario: Checkout rates use vendor-specific origin address
- **Given** a customer adds items from Vendor A (Lisbon, Portugal) and Vendor B (Madrid, Spain) to their cart and proceeds to checkout
- **When** the checkout page calculates shipping rates
- **Then** the rates for Vendor A's items use Lisbon as the origin and the rates for Vendor B's items use Madrid as the origin, with each set of rates fetched from the respective vendor's DHL Express account
