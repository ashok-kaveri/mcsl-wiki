## User Story

As a **UK merchant using Sage accounting and WooCommerce with tax-inclusive shipping zones**,
I want **the plugin to pass FedEx's base rate (excluding VAT) to WooCommerce so WooCommerce can calculate VAT natively**,
so that **my customers aren't charged VAT twice on shipping, my accounting reports in Sage show accurate VAT breakdown, and I can use FedEx rates without manually overriding or disabling WooCommerce tax calculation**.

---

## Acceptance Criteria (Simple)

- [ ] Plugin retrieves `totalNetFedExCharge` (VAT-exclusive rate) from FedEx API and passes only this value to WooCommerce for rate display
- [ ] When WooCommerce applies native tax calculation on FedEx rates, customers see correct single VAT charge (not double)
- [ ] alexisfilters.co.uk store can enable WooCommerce tax on shipping, select FedEx rate, and verify checkout shows base rate + single VAT charge
- [ ] WooCommerce order reports and Sage accounting exports show separate shipping base and VAT amounts (not combined)
- [ ] Configuration option added to plugin settings: "Display FedEx rates excluding VAT/taxes" (default: enabled for VAT regions)

---

## Acceptance Criteria (Given/When/Then)

### Scenario: UK merchant uses FedEx rates with WooCommerce native tax
- **Given** alexisfilters.co.uk has WooCommerce tax enabled for shipping (UK VAT 20%)
- **And** customer has enabled "Display FedEx rates excluding VAT/taxes" setting
- **When** customer selects a FedEx shipping option at checkout
- **Then** WooCommerce displays base FedEx rate (£10.00) + separately calculated VAT (£2.00) = £12.00 total
- **And** cart does NOT show £12.00 + £2.40 VAT = £14.40 (double taxation)
- **And** order confirmation email and invoice show shipping £10.00 and shipping tax £2.00 as separate line items

### Scenario: Cyprus merchant with multi-country VAT requirements (ticket #385094)
- **Given** Cyprus merchant ships to multiple EU countries with different VAT rates
- **And** WooCommerce is configured to apply destination-country VAT on shipping
- **When** customer from Germany selects FedEx shipping (19% VAT)
- **Then** WooCommerce applies German VAT to the base FedEx rate (not UK VAT)
- **And** customer sees correct German VAT amount, not double-taxed with FedEx's UK VAT + German VAT

### Scenario: Merchant exports to Sage without manual VAT separation
- **Given** alexisfilters.co.uk exports WooCommerce orders to Sage accounting
- **When** merchant generates VAT report for HMRC filing
- **Then** Sage report shows accurate shipping revenue (net) and VAT collected (tax) in separate columns
- **And** merchant does NOT need to manually split combined FedEx charges into base + VAT for accounting compliance
