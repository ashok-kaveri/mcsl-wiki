# Ticket #392107 — DHL Express estimated delivery showing incorrect range at checkout

- **Customer**: Matt (95a96e-3.myshopify.com)
- **Product**: Shopify Multi Carrier Shipping Label
- **Duration**: 2026-05-26 to 2026-06-02 (7 days)

## Timeline & Key Phases

### Phase 1: Initial Inquiry (2026-05-26)
- Customer reports DHL Express showing "1-2 business days" at checkout when DHL portal only shows 2 days minimum for US orders
- Support (Sagar) explains: transit time comes directly from DHL API based on origin/destination
- Customer requests two solutions: disable the display or add ability to append additional days

### Phase 2: Investigation & Evidence Gathering (2026-05-28 to 2026-06-02)
- Customer provides detailed screenshots confirming DHL portal consistently shows "2 days" for multiple US shipments
- Customer asks: why does app show "1-2 days" when DHL only provides "2 days"? Where does the "1" come from?
- Customer suggests alternative: use DHL's "Est. delivery date" field instead of "Total transit days"
- Support (Sagar) checks transaction log; confirms app received "4 days" from DHL API
- Support (Touseef) clarifies: app passes exact delivery date to Shopify; Shopify applies its own business day calculation and format

### Phase 3: Root Cause Identification (2026-06-02)
- **Root Cause**: Shopify's checkout display format is controlled by Shopify, not by the app
- Shopify converts carrier transit days into a "1 to N" business days range format
- The "1 to" prefix is added by Shopify's standard format, not by the MCSL app
- App passes exact transit days to Shopify correctly

## Open Issues

## User Story

As an **Australian Shopify merchant whose customers see "1-2 business days" for DHL Express at checkout — because Shopify reformats the carrier's transit estimate into a range — while DHL's own portal consistently shows 2+ days for the same route**,
I want **a per-carrier "buffer days" setting that adds fulfillment and handling time on top of the carrier's transit estimate before it reaches Shopify's checkout display**,
so that **my customers get realistic delivery expectations, I stop fielding complaints from buyers who expected next-day delivery that DHL can't fulfill, and I don't have to choose between showing inaccurate fast estimates or hiding delivery dates entirely**.

---

## Acceptance Criteria (Simple)

- [ ] Per-carrier "buffer days" setting available in MCSL carrier configuration (integer, 0-10 range)
- [ ] Buffer days are added to the carrier API's transit time before passing to Shopify checkout
- [ ] Setting is carrier-specific — DHL can have +1 day while FedEx stays at 0
- [ ] Default is 0 for all carriers (backward compatible, no behavior change for existing merchants)
- [ ] Matt's Australian store confirms DHL checkout estimate aligns with actual DHL portal transit times after adding buffer

---

## Acceptance Criteria (Given/When/Then)

### Scenario: Buffer days added to DHL estimate
- **Given** a merchant has DHL Express configured with buffer_days = 1
- **When** DHL API returns "2 business days" for a Sydney → Melbourne shipment
- **Then** MCSL passes "3 business days" to Shopify checkout
- **And** Shopify displays "2-3 business days" (its standard range format)

### Scenario: Different carriers use different buffers
- **Given** DHL buffer = 1, FedEx buffer = 0, Australia Post buffer = 2
- **When** rate shopping returns estimates for all three carriers
- **Then** each carrier's displayed estimate includes only its own buffer
- **And** FedEx shows raw carrier estimate unchanged

### Scenario: Zero buffer preserves existing behavior
- **Given** a merchant has not configured any buffer days (default = 0)
- **When** DHL API returns "2 business days"
- **Then** MCSL passes "2 business days" to Shopify unchanged
- **And** checkout display is identical to pre-feature behavior
