## Timeline & Key Phases

### Phase 1: Initial Package Dimensions Query (2026-04-21)
- Customer inquired about inability to change FedEx package dimensions (minimum 20x20x20)
- Support clarified dimensions are only for freight shipments (>150 lbs), not normal shipments
- Customer then requested FedEx request/response logs showing "N/A" for rates

### Phase 2: Cyprus FedEx Rates Investigation (2026-04-21 - 2026-04-22)
- Customer reconnected FedEx account but request/response still showing N/A despite rates displaying
- Support escalated to L3 (Slack thread created)
- Engineering initially determined Cyprus not supported by FedEx API, rates were "dummy rates"
- Advised customer to use DHL rates or flat rates instead

### Phase 3: FedEx Representative Intervention (2026-04-23)
- Customer added Emmanouil from FedEx who confirmed Cyprus was enabled in Q4 2025
- FedEx rep identified issue: invalid postal code (10565 for Greece) in test request
- With valid postal code (19003), FedEx rates returned successfully
- Support enabled live FedEx rates for the store

### Phase 4: VAT Double Taxation Issue Discovery (2026-04-23 - 2026-04-24)
- Customer identified FedEx rates include Cyprus VAT (19%) in the rate response
- WooCommerce also applies destination country VAT (e.g., Greece 24%, Germany 19%, Sweden 25%) on top
- Customer explained EU eCommerce tax rules require charging only destination country VAT
- Customer requested option to use `totalNetFedExCharge` (excluding VAT) similar to DHL carrier settings
- Support initially stated taxes cannot be excluded from FedEx response
- Customer provided detailed analysis showing FedEx API returns both with-VAT and without-VAT rates

### Phase 5: Feature Enhancement Approved (2026-04-27)
- Engineering approved enhancement to display FedEx rates without VAT/taxes at checkout
- Prioritized for release 378 (tentatively third week of May 2026)
- Customer acknowledged and waiting for release

## User Story

As a **Cyprus-based EU merchant shipping to multiple EU countries with different VAT rates**,
I want **FedEx shipping rates displayed at checkout to exclude origin-country VAT so WooCommerce can apply only the correct destination-country VAT**,
so that **my customers aren't charged VAT twice on shipping (19% Cyprus VAT + 24% Greece VAT), my eCommerce tax compliance stays intact, and I don't have to abandon FedEx and use only DHL or abandon the plugin entirely to generate FedEx labels manually on the carrier portal**.

---

## Acceptance Criteria (Simple)

- [ ] Plugin adds a FedEx-specific setting "Display rates excluding VAT" (similar to existing DHL setting)
- [ ] When enabled, plugin uses `totalNetFedExCharge` (net rate without VAT) instead of `totalNetFedExCharge + totalVatCharge`
- [ ] WooCommerce correctly applies destination country VAT percentage on the net shipping rate
- [ ] Customer in Cyprus ships test order to Greece: checkout shows FedEx rate with only Greek VAT (24%), not double-taxed
- [ ] All EU merchants with multi-country VAT setups can use FedEx rates without tax compliance violations

---

## Acceptance Criteria (Given/When/Then)

### Scenario: Cyprus merchant ships to Greece with FedEx (Happy Path)
- **Given** Cyprus store (19% VAT) with FedEx integration enabled
- **And** "Display rates excluding VAT" setting enabled for FedEx
- **And** Customer shipping address is in Greece (24% VAT)
- **When** Customer checks out with a FedEx service
- **Then** Checkout displays FedEx base rate (e.g., €15.00) + Greek VAT 24% (€3.60) = €18.60 total
- **And** Cyprus VAT (19%) is NOT applied to the shipping rate

### Scenario: Broader Fix - Works for all EU origin countries
- **Given** Merchant in any EU country (Germany 19%, France 20%, Sweden 25%, etc.)
- **And** "Display rates excluding VAT" setting enabled for FedEx
- **When** Customer from a different EU country checks out
- **Then** Only destination country VAT is applied to FedEx shipping rate
- **And** No double taxation occurs regardless of origin/destination VAT rate combination

### Scenario: Prevention - Setting is discoverable and documented
- **Given** New EU merchant installing WooCommerce plugin with FedEx integration
- **When** Merchant navigates to FedEx carrier settings
- **Then** "Display rates excluding VAT" checkbox is visible with clear label
- **And** Tooltip or help text explains: "For EU merchants: Enable to prevent double VAT taxation. Plugin will use net FedEx rates, allowing WooCommerce to apply correct destination VAT."
- **And** Setting defaults to OFF for backward compatibility with non-EU merchants
