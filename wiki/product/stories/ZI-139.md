## User Story

As a **$29-plan Shopify merchant using MCSL daily for Blue Dart labels across 133 orders, blocked from shipping Order 6836 because it simply doesn't appear in the app**,
I want **to see why a specific order isn't showing up — and to surface or recover it myself without waiting days for a support agent to get collaborator access to my store**,
so that **my customer waiting on Order 6836 gets their package on time, and I have confidence that I'll know immediately if any future order silently drops out of MCSL rather than discovering it from a customer complaint**.

## Acceptance Criteria (Simple)

- [ ] Order 6836 on tss.digital's store is visible in MCSL's "Orders to Ship" and a Blue Dart label can be generated for it
- [ ] The root cause (order property filter / fulfillment-status filter / sync failure / other) is identified and documented, and the fix covers all orders with the same profile — not just 6836
- [ ] The Orders view shows a "missing orders" diagnostic: merchant can enter any Shopify Order ID and see either "imported (here's the link)" or "not imported — reason: X" without contacting support
- [ ] Import logs / sync errors for specific orders are surfaced in-app (per-order, timestamped) so merchants and support can triage without needing collaborator access to Shopify
- [ ] If an order is excluded by a filter (tag, fulfillment status, line-item type), MCSL shows the specific filter rule that excluded it so merchants can toggle it off themselves

## Acceptance Criteria (Given/When/Then)

### Scenario: Recover Order 6836 for tss.digital
- **Given** Order 6836 exists in tss.digital's Shopify store but is not visible in MCSL
- **When** the import-diagnostic flow is run against that order ID
- **Then** MCSL reports the specific reason (fulfillment status / tag filter / sync error / other)
- **And** the merchant can either toggle the filter off or trigger a targeted re-sync, and the order appears in Orders to Ship with Blue Dart as an option

### Scenario: Cover all similar hidden orders, not just this one
- **Given** a merchant's store has N total Shopify orders and M of them are in MCSL
- **When** the reconciliation diagnostic runs
- **Then** the merchant sees the exact count of missing orders, grouped by exclusion reason
- **And** each group has a bulk action (e.g., "Include 12 orders filtered by tag X", "Re-sync 3 orders with sync errors")

### Scenario: Remove collaborator-access as a prerequisite for basic diagnosis
- **Given** a merchant reports a missing-order issue
- **When** support runs the self-serve diagnostic
- **Then** 80%+ of cases can be resolved without collaborator access
- **And** the remaining cases show a clear, narrower reason that requires collaborator access, with a one-click re-invite link
