## User Story

As an **India-based merchant on the $29 plan processing 235 orders, currently paying BlueDart rates but wanting access to India Post's government-subsidized domestic shipping rates which are 30-40% cheaper for most pin codes**,
I want **transparent status updates on whether India Post integration is technically feasible, with a clear timeline (2 weeks for assessment, 4-6 weeks for development if approved) and upfront honesty if PluginHive can't support it due to API limitations or business priorities**,
so that **I don't wait 6 days wondering if anyone is actually reviewing the API docs I provided, can make an informed decision whether to stick with BlueDart or switch to a competitor that already supports India Post, and don't feel like I wasted time gathering documentation and technical contacts for an integration that was never going to happen**.

## Acceptance Criteria (Simple)

- [ ] Carrier integration request form captures: carrier name, country, customer's shipping volume, API documentation URL, business case (why this carrier matters)
- [ ] Auto-response email sets expectations: "Feasibility assessment: 5 business days. We'll evaluate API complexity, volume potential, and strategic fit. You'll receive a decision by [date]."
- [ ] Engineering reviews API docs within 5 days and provides one of three outcomes: "Approved - ETA 6-8 weeks", "Needs more info - questions attached", or "Not feasible - here's why"
- [ ] If approved: customer receives implementation plan with milestones (API sandbox access, rate testing, label testing, production rollout)
- [ ] If rejected: customer receives detailed explanation (API limitations, business case too niche, alternative suggestions) and 10% discount code for 3 months as goodwill

## Acceptance Criteria (Given/When/Then)

### Scenario: Customer submits carrier integration request with documentation
- **Given** merchant wants India Post integration and has API documentation
- **When** merchant submits carrier integration request form with India Post API docs link
- **Then** auto-response email confirms receipt and sets expectation: "We'll assess feasibility by [5 business days from now]. L3 team will review API complexity and market demand."
- **And** ticket is auto-assigned to carrier integrations queue with "5-day SLA" tag

### Scenario: Engineering completes feasibility assessment (approved)
- **Given** L3 reviewed India Post API documentation
- **And** determined integration is technically feasible and strategically valuable
- **When** assessment is complete (within 5 days)
- **Then** customer receives email: "Good news! India Post integration approved. Timeline: 6-8 weeks. Milestones: Week 2 (sandbox testing), Week 4 (rate accuracy validation), Week 6 (label generation beta), Week 8 (production release). We'll update you at each milestone."

### Scenario: Engineering completes feasibility assessment (not feasible)
- **Given** L3 reviewed India Post API but found it lacks label generation endpoints (only tracking available)
- **When** assessment is complete
- **Then** customer receives honest explanation: "We reviewed India Post's API. Unfortunately, it only supports tracking, not label generation or rate quotes. This means we can't offer the full shipping workflow you need. Alternative: Continue using BlueDart, or consider Shiprocket which has an India Post partnership. As a thank you for your time gathering docs, here's 10% off for 3 months."
