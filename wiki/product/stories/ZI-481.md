## Open Issues

1. **PostNord international label and CN22 must print on single page instead of separate PDFs** — Customer requires shipping label and CN22 customs document to print on one sheet for international orders (standard PostNord workflow). Currently prints as two separate documents. Blocked: customer (uninstalled app, awaiting development). Severity: P1 blocker (prevents go-live for 100 orders/month). Area: international. (Comments #52, #60, #61)

2. **PostNord return service codes not implemented** — Customer provided specific PostNord return product codes and transport codes directly from PostNord. Need to implement these service codes for return label generation. Blocked: engineering (pending as part of broader PostNord enhancement). Severity: P2 (part of full implementation). Area: carrier-config. (Comments #34, #40, #42, #60)

## Resolved Issues

1. **PostNord "For US customs categoryType is mandatory" error on label generation** — Resolved by updating Category Type under Settings → Carrier → PostNord → Other Details and updating currency configuration. Label successfully generated for order SE11912. (Comment #10, 2026-03-23)

2. **Country-specific HS codes not supported** — Customer required different HS codes for different destination countries (standard requirement for international shipping). Development team enabled custom Country-wise HS Code feature via L3 escalation. Webhooks resynced and products force-imported. Feature demonstrated on call Mar 26. (Comments #41, #43, #44, 2026-03-25)
