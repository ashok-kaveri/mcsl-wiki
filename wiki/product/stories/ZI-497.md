## User Story

As a **Canadian merchant on a Yearly Starter plan using YayCommerce's YayCurrency switcher** (already integrated with my theme), I want **WSS to convert shipping fees into the customer's selected display currency** the same way it already does for Aelia and the official WooCommerce Currency Switcher, so that **my checkout doesn't show a mismatched fee in the wrong currency next to YayCurrency-converted product prices, and I don't have to migrate currency plugins just to keep using WSS for live rates**.

---

## Acceptance Criteria (Simple)

- [ ] Customer's YayCurrency-configured store displays shipping fees converted into the selected display currency at checkout (matching the product-price currency)
- [ ] Conversion uses the merchant-configured YayCurrency exchange rate (no live FX dependency on PluginHive side); merchant changes propagate without WSS-side cache invalidation
- [ ] Existing compatibility with Aelia Multi-currency Switcher and official WooCommerce Currency Switcher is preserved (no regression)
- [ ] Documentation lists YayCurrency as a supported switcher in the compatibility matrix
- [ ] Customer can switch display currency on somtico.com and see WSS shipping fees follow without any per-order workaround

---

## Acceptance Criteria (Given/When/Then)

### Scenario: Customer switches to CAD on a YayCurrency-enabled store

- **Given** a Canadian shopper on somtico.com with YayCurrency configured for USD↔CAD
- **When** the shopper switches the display currency to CAD and views the shipping fee at checkout
- **Then** WSS converts the shipping fee using the merchant-configured YayCurrency rate and displays it in CAD
- **And** the converted fee matches what is charged at checkout (no currency-mismatch between display and capture)

### Scenario: No regression on Aelia / official Woo Currency Switcher

- **Given** a different store on Aelia Multi-currency Switcher
- **When** shipping fees are rendered at checkout in any non-base currency
- **Then** behavior is identical to today — Aelia and official switcher continue working unchanged
- **And** a regression test in the Aelia integration suite continues to pass

### Scenario: Customer adopts WSS without changing their currency-switcher plugin

- **Given** any future customer using YayCurrency
- **When** they enable WSS for the first time
- **Then** they can complete onboarding without being told 'YayCurrency is untested, switch to a different plugin'
- **And** the compatibility matrix in the WSS docs lists YayCurrency explicitly with the supported version range
