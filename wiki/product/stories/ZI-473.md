## User Story

As a **WooCommerce merchant who has been using the plugin successfully with UPS but suddenly cannot generate any USPS labels due to a cryptic payment authorization error I didn't cause**,
I want **clear, actionable error messages that tell me exactly what's wrong with my USPS account setup and how to fix it, OR automatic detection and handling of common USPS API credential issues**,
so that **I don't waste days opening support tickets and waiting for L3 investigation just to discover I need to call USPS directly about an API payment authorization setting I've never heard of, while my orders pile up and I'm forced to manually process USPS shipments outside the app I'm paying $30/month for**.

## Acceptance Criteria (Simple)

- [ ] USPS label generation errors include specific, actionable error messages (not just a hash like "03d7a45e9f42...unavailable")
- [ ] When USPS API returns 401 payment authorization error, plugin displays clear message: "USPS payment authorization failed. Please verify your USPS account settings or contact USPS support at [phone]"
- [ ] Plugin proactively validates USPS payment authorization when account is first connected, before customer tries to generate first label
- [ ] Customer Michelle's store (ticket #302656) can generate USPS labels without 401 errors after following error message guidance
- [ ] Support documentation includes troubleshooting guide for USPS 401 payment authorization errors with step-by-step resolution

## Acceptance Criteria (Given/When/Then)

### Scenario: Clear error message when USPS payment auth fails
- **Given** a WooCommerce store has USPS configured in the plugin
- **And** the USPS account has a payment authorization issue (401 error from USPS API)
- **When** the merchant attempts to generate a USPS label for an order
- **Then** label generation fails with a user-friendly error message: "USPS Payment Authorization Error: Your USPS account's payment settings could not be verified. Please contact USPS at 1-800-344-7779 to verify your account payment authorization is active."
- **And** the error message includes a link to plugin documentation with troubleshooting steps
- **And** the error is logged with the full USPS API response for support team investigation

### Scenario: Proactive validation during USPS setup
- **Given** a merchant is connecting their USPS account in the plugin for the first time
- **When** they save their USPS credentials
- **Then** the plugin immediately tests payment authorization by making a test API call to USPS
- **And** if the test fails with 401 error, displays warning: "Warning: USPS payment authorization check failed. You may not be able to generate labels. Please verify your USPS account settings."
- **And** the merchant can still save the credentials but sees this warning prominently

### Scenario: Existing accounts detect auth issues before label generation
- **Given** a merchant has been using USPS successfully but their payment auth expires or is revoked
- **When** they navigate to the orders page or attempt label generation
- **Then** the plugin displays a banner alert: "USPS Payment Authorization Issue Detected: Your USPS credentials may need to be refreshed. Click here to re-verify."
- **And** clicking the alert takes them to USPS settings with validation instructions
- **And** they don't discover the issue only after a customer order fails
