# Ticket #391779 — Rate automation not triggering for Guam and Marshall Islands shipping zones

- **Customer**: Ankur Verma (laurasgourmet.com)
- **Product**: Shopify Multi Carrier Shipping Label

## Timeline & Key Phases

### Phase 1: Initial Setup & Configuration (2026-05-23 to 2026-05-25)
- Customer configured rate automation for CounterCool product with two shipping zones:
  - Zone 1: Contiguous US + AK + HI → $15 flat rate per unit
  - Zone 2: PR + Guam + Marshall Islands + AA → $20 flat rate per unit
- Tested on production site, reverted to avoid customer-facing errors
- Reported that checkout shows "no shipping available" error for PR, Guam, Marshall Islands, AA

### Phase 2: Support Troubleshooting (2026-05-23 to 2026-05-25)
- Support added PR as "US-Puerto Rico" option in shipping zone UI → rates appeared for PR
- Support added AA as "US-AA" option in shipping zone UI → rates appeared for AA
- **Gap identified**: No "US-Guam" or "US-Marshall Islands" options available in shipping zone dropdown
- Support suggested adding Guam/Marshall Islands to existing PR zone, but customer rejected (would show wrong $15 rate)

### Phase 3: L3 Investigation & Root Cause (2026-05-25 to 2026-05-27)
- L3 (Veena) granted collaborator access to investigate
- Support removed and re-added MCSL app rates from shipping profile
- L3 captured request log for Guam address check (ID: aa7bb900c0458a02804875b0b83872e4)
- **Root cause identified** (Comment #17): Automation rule is not triggering for Guam addresses. Automation summary in request log is blank.
- L3 decision: "This needs a fix" — scheduled for next cycle. Recommended workaround: use Shopify's native flat rate for Guam/Marshall Islands.

### Phase 4: Workaround Deployed (2026-05-27)
- Customer disabled $20 flat rate automation rule
- Removed special shipping zones from $15 rule
- Currently offering $15 rate to all destinations as interim solution
- Awaiting fix in next development cycle (approx. 2 weeks)

## User Story

As a **specialty food retailer shipping gourmet products to all US territories including Guam and Marshall Islands, who configured automation rules to charge $20 for outlying territories vs $15 for contiguous US — and whose customers in Guam now see "no shipping available" at checkout because the rule fails silently**,
I want **the rate automation engine to recognize Guam, Marshall Islands, and all US territory addresses by adding them to the shipping zone configuration dropdown**,
so that **I stop losing sales from customers who abandon checkout when no shipping option appears, and I can retire the $15 flat-rate-everywhere workaround that's been eating my margin on every territory order since the rule broke**.

## Acceptance Criteria (Given/When/Then)

### Scenario: Guam address gets territory rate
- **Given** an automation rule "If destination is US outlying territory, charge $20 flat rate"
- **When** a customer in Hagatna, Guam 96910 checks out
- **Then** checkout displays "$20.00 — Flat Rate Shipping"
- **And** request log shows automation rule matched with populated summary

### Scenario: Marshall Islands recognized
- **Given** the same $20 territory rule is active
- **When** a customer in Majuro, Marshall Islands 96960 checks out
- **Then** the $20 flat rate applies (not "no shipping available")
- **And** the zone configuration shows "US-Marshall Islands" as matched zone

### Scenario: Contiguous US unaffected
- **Given** both $15 contiguous and $20 territory rules active
- **When** a customer in Austin, TX 78701 checks out
- **Then** $15 flat rate applies (contiguous rule matches)
- **And** no regression in existing US-48 rate behavior
