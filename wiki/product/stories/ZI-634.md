## User Story

As a **partnerships lead managing the PluginHive x PostNord Scandinavia carrier integration**,
I want **PostNord's Delivery Options API surfaced at Shopify checkout so merchants see real-time delivery dates, time windows, and service options from PostNord**,
so that **we fulfill our partnership commitment by the July 7 deadline and Scandinavian merchants stop losing checkout conversions to opaque "flat shipping" labels that hide PostNord's actual delivery promise**.

---

## Acceptance Criteria (Simple)

- [ ] PostNord Delivery Options API is callable from the app using sandbox credentials, returning available delivery options for a given address and parcel
- [ ] Checkout displays PostNord delivery options with estimated delivery date and latest delivery time window per service
- [ ] Merchants with PostNord enabled see DO-sourced options without additional configuration beyond existing PostNord carrier setup
- [ ] The integration gracefully degrades when the DO API is unreachable — falls back to static PostNord services without breaking checkout
- [ ] Partnership team (Abhilash S) is notified upon deployment completion, per the internal handoff agreement in Comment #8
- [ ] PostNord Rating API absence is documented as a known limitation — rate-shopping for PostNord is deferred until PostNord ships their Rating API

---

## Acceptance Criteria (Given/When/Then)

### Scenario: PostNord Delivery Options displayed at checkout
- **Given** a Shopify merchant has PostNord enabled as a carrier and a customer enters a Scandinavian delivery address at checkout
- **When** the checkout fetches shipping rates
- **Then** the response includes PostNord delivery options with service name, estimated delivery date, and latest delivery time window sourced from the DO API

### Scenario: Graceful fallback when DO API is unavailable
- **Given** a merchant has PostNord enabled but the DO API returns an error or times out
- **When** a customer reaches checkout
- **Then** the app falls back to existing static PostNord service options without surfacing an error to the customer and logs the failure for monitoring

### Scenario: No rate-shopping for PostNord until Rating API ships
- **Given** a merchant has rate-shopping enabled across multiple carriers including PostNord
- **When** rate requests are made at checkout
- **Then** PostNord delivery options are displayed alongside other carrier rates, but PostNord rates are not price-compared until the Rating API becomes available, and the merchant sees a clear indicator that PostNord pricing is not live-quoted
