## User Story

As a **merchant on the $29 Popular plan trying to ship order #18501 right now**,
I want **label generation to never fail with a cryptic "Amount is a required field" error that I have no way to fix myself**,
so that **I don't have to open a support ticket and wait 6+ days just to ship one order while my customer is already wondering why their shoes from Sweden haven't moved**.

---

## Acceptance Criteria (Simple)

- [ ] Order #18501 for lescarf.myshopify.com generates a PostNord or FedEx label without the "Amount is a required field" error
- [ ] The missing "Amount" field is identified — which entity (item, package, customs, insurance, declared value) and which carrier path triggers the validation
- [ ] When the field is genuinely missing from the order data, the UI surfaces a plain-English message naming the exact field and an inline way to fill it in — not a raw API validation string
- [ ] A pre-flight validation on the order grid flags this class of missing-field error before the merchant clicks Generate, so they never see the cryptic error mid-shipment
- [ ] Giuseppe is contacted with the root cause and resolution, plus an apology for the 6-day SLA breach on an active-paying $29/mo store

---

## Acceptance Criteria (Given/When/Then)

### Scenario 1: Order #18501 ships successfully
- **Given** Giuseppe opens order #18501 at lescarf.myshopify.com
- **When** he clicks Generate Label with either PostNord or FedEx selected
- **Then** a valid label is produced and the tracking number syncs back to Shopify
- **And** no "Amount is a required field" error appears

### Scenario 2: Missing-field errors are human-readable for any merchant
- **Given** any merchant has an order where a required carrier field (amount, weight, declared value, etc.) cannot be derived from the Shopify order data
- **When** they attempt label generation
- **Then** the UI shows which field is missing, where it comes from, and how to enter it
- **And** the carrier's raw API error is logged for support but not shown directly to the merchant

### Scenario 3: Pre-flight catches these before shipment time
- **Given** a batch of orders is queued for label generation
- **When** the pre-flight validator runs
- **Then** orders missing required carrier fields are flagged with the specific gap and separated from the ready-to-ship set
- **And** the merchant fixes the flagged orders before the carrier API is ever called, preventing partial failures mid-batch
