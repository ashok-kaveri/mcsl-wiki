## User Story

As a **US-based Shopify merchant who ships LTL freight and has a contracted account with TForce Freight (formerly UPS Freight)**,
I want **MCSL to support TForce as a first-class carrier on par with how it supports FedEx, UPS, and USPS — rate quoting, label printing, BOL generation, pickup scheduling — so I can stop juggling carriers across two systems**,
so that **I can consolidate my entire shipping stack (parcel + freight) in one app, drop the spreadsheet I currently use to manually generate TForce labels on their portal after MCSL handles my parcel, and finally bring my freight margin into the same automation rules I already run for parcel — three other merchants have been asking for this for months and at least one has hinted they'll leave MCSL if it doesn't land**.

---

## Acceptance Criteria (Simple)

- [ ] TForce Freight appears in MCSL's "Add carrier" list with the same connection UX as the other supported US carriers (account number, API credentials, terms-of-service link, test/live mode toggle)
- [ ] A merchant with a valid TForce account can connect it inside MCSL in under 10 minutes following on-screen docs, with no support ticket required
- [ ] Once connected, the merchant can fetch live LTL rates for an order, generate a TForce shipping label (and the carrier's required BOL), and the label/tracking number write back into the order grid like any other carrier
- [ ] All three original requester tickets (#270058, #263693, #274726) are notified when TForce ships, with a direct setup link — proving the integration closed the loop on the customers who asked
- [ ] Automation rules can trigger off TForce-specific criteria (weight thresholds, freight class, accessorials like liftgate / residential) so merchants can mix-and-match TForce with parcel carriers in the same auto-routing logic

---

## Acceptance Criteria (Given/When/Then)

### Scenario: Customer who originally asked for TForce can self-serve onboard once it ships
- **Given** a merchant from one of the three original requester tickets (#270058, #263693, #274726) opens MCSL after the TForce integration releases
- **When** they click "Add carrier" and select TForce Freight
- **Then** they can connect their TForce account using only the docs in the connection panel, see live LTL rates against a sample order within 10 minutes, and generate a working test label + BOL — all without opening a new support ticket

### Scenario: Mixed parcel + freight order works in a single label batch
- **Given** a merchant with both TForce and FedEx connected, processing a batch where some orders qualify for parcel and others trigger LTL freight rules (weight > 150 lb or freight class specified)
- **When** the merchant runs the batch through MCSL's automation rules and label generation
- **Then** each order ships on the carrier its rule selected — parcel orders go FedEx, LTL orders go TForce — and both produce valid labels/BOLs, tracking numbers, and order updates in the same run without manual intervention

### Scenario: TForce credentials revoked or rate API down doesn't break parcel
- **Given** a merchant has TForce + parcel carriers connected, and TForce's API returns persistent 5xx errors or auth failures for an hour
- **When** the merchant generates labels for the day's batch
- **Then** orders rule-routed to TForce show a clear "TForce: account temporarily unreachable, retry later" error per-row, but every parcel-bound order in the same batch still ships normally — TForce instability does not cascade into a global label-generation failure
