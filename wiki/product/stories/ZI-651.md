## User Story

As a **WooCommerce merchant who frequently corrects order details (addresses, line items) after import and occasionally reprocesses cancelled orders**,
I want **order edits made in WooCommerce to automatically propagate to WSS regardless of the order's status transition history (including cancelled-then-reprocessed)**,
so that **I don't ship to stale addresses or with wrong items, which causes costly mis-deliveries and return handling for my titanium parts business**.

---

## Acceptance Criteria (Simple)

- [ ] When a WooCommerce order transitions from Cancelled to Processing, the WSS order record re-activates and reflects the current WooCommerce state
- [ ] Subsequent edits to a cancelled-then-reprocessed order (address, line items, quantities) sync to WSS within the same timeframe as normal orders (~1 minute)
- [ ] The order update webhook handler does not skip orders whose previous status was Cancelled
- [ ] Existing auto-sync behavior for orders that were never cancelled remains unaffected (regression guard)
- [ ] WSS order grid shows the latest WooCommerce data after a cancel-reprocess-edit cycle without manual intervention

---

## Acceptance Criteria (Given/When/Then)

### Scenario: Cancelled order reprocessed and edited syncs to WSS
- **Given** a WooCommerce order was imported into WSS, then cancelled in WooCommerce, and later moved back to Processing
- **When** the merchant edits the shipping address on that reprocessed order in WooCommerce
- **Then** the updated shipping address appears in WSS within 60 seconds without manual re-import

### Scenario: Line item changes on reprocessed order propagate correctly
- **Given** a WooCommerce order was cancelled, reprocessed, and then has line items added or removed in WooCommerce
- **When** the WooCommerce webhook fires the order update event for the modified order
- **Then** WSS reflects the updated line items, quantities, and weights matching the current WooCommerce order state

### Scenario: Normal order sync unaffected by cancel-reprocess fix
- **Given** a WooCommerce order that has never been cancelled and is in Processing status
- **When** the merchant edits the order's shipping address in WooCommerce
- **Then** the update syncs to WSS within 60 seconds, confirming no regression in the standard sync path
