## User Story

As a **WooCommerce merchant who ships via USPS flat rate boxes and relies on accurate rate quotes to price my shipping correctly**,
I want **USPS flat rate box dimensions in the plugin to match actual USPS specifications, and packing logic to correctly fit products into appropriate box sizes**,
so that **I don't lose money by offering inaccurate shipping rates to customers, and my orders pack efficiently without manual intervention to fix dimension mismatches that cause rate calculation errors**.

## Acceptance Criteria (Simple)

- [ ] All USPS Flat Rate box dimensions (Small, Medium, Large, Regional A/B) match official USPS specifications exactly
- [ ] Kenny's WooCommerce store (#364411) can select USPS Flat Rate boxes without dimension validation errors

- [ ] Automated tests validate USPS Flat Rate dimensions against USPS API documentation to prevent future mismatches
- [ ] Rate calculator uses correct Flat Rate pricing for each box size (not standard parcel rates)

## Acceptance Criteria (Given/When/Then)

### Scenario: Merchant selects correct USPS Flat Rate box with accurate dimensions
- **Given** a WooCommerce store has products that fit in USPS Flat Rate Small box (8-11/8" x 5-3/8" x 1-5/8")
- **When** the merchant generates a label and selects "USPS Flat Rate Small" as packaging
- **Then** the system accepts the box dimensions without validation errors
- **And** the shipping label shows correct Flat Rate Small pricing
- **And** the packing slip indicates the correct box size

### Scenario: All USPS Flat Rate box sizes have accurate dimensions
- **Given** the plugin has configuration for all USPS Flat Rate boxes (Small, Medium, Large, Regional A/B)
- **When** a merchant selects any Flat Rate box option
- **Then** the dimensions match official USPS specifications exactly
- **And** packing logic correctly determines if products fit within those dimensions
- **And** rate calculation uses the correct Flat Rate pricing for that box size

### Scenario: Dimension validation prevents future mismatches
- **Given** USPS updates their Flat Rate box specifications
- **When** the plugin dimensions are updated to match
- **Then** automated tests validate dimensions against USPS API documentation
- **And** any dimension mismatch triggers a build failure
- **And** merchants are notified if active configurations need updating
