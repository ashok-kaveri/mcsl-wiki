## Acceptance Criteria (Given/When/Then)

### Scenario: Merchant connects Emirates Post account
- **Given** a Shopify MCSL merchant navigates to Carrier Settings and selects Emirates Post
- **When** they enter their EMX API credentials (API Key, Site ID) and click Connect
- **Then** the system validates the credentials against the EMX staging endpoint and shows a successful connection status

### Scenario: Rate calculation for domestic UAE shipment
- **Given** a connected Emirates Post account with domestic product codes configured
- **When** the merchant requests shipping rates for an order with a UAE origin and UAE destination address
- **Then** the system calls the EMX Rate Calculation API and returns available domestic service options with prices

### Scenario: Label generation creates shipment and returns AWB
- **Given** a connected Emirates Post account and an order ready to ship
- **When** the merchant generates a shipping label selecting an EMX domestic product code
- **Then** the system calls `POST /api/Shipments/create`, stores the returned AWB number, and renders a downloadable label

### Scenario: Tracking returns shipment status
- **Given** a shipment created via Emirates Post with a valid AWB number
- **When** the merchant views tracking details for that shipment
- **Then** the system queries the EMX Tracking API and displays the current shipment status and event history

### Scenario: International and GCC product codes supported
- **Given** a connected Emirates Post account
- **When** the merchant selects an international or GCC destination
- **Then** the rate calculation and shipment creation use the appropriate EMX international/GCC product codes (including DDP/DDU variants)
