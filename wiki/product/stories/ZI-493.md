## Timeline & Key Phases

### Phase 1: L3 Root Cause Documentation (2026-04-29)
- Ticket created as **alternate/tracking ticket** for #383435
- L3 identified hidden setting in PluginHive settings page causing checkout rates to fail
- Root cause: Hidden boolean setting defaults to `true`, but for some merchants it gets set to `false`
- This hidden setting is redundant — app already has "RealTime Rates" setting for displaying rates
- No customer interaction documented; internal L3 diagnostic ticket only

## Open Issues

1. **Hidden redundant setting causing checkout rate failures** — WooCommerce plugin has a hidden boolean setting (not exposed in UI) that controls rate display, separate from the visible "RealTime Rates" setting. For some merchants, this hidden setting gets flagged as `false`, causing checkout rates to not display despite RealTime Rates being enabled. L3 states this hidden setting is unnecessary and should be removed. Blocked: engineering (need to remove redundant setting and migrate affected merchants). Severity: High (breaks core functionality — rates not showing at checkout). Area: `rate-shopping`. (Comment #1)

## User Story

As a **WooCommerce store owner who enabled RealTime Rates to display carrier quotes at checkout**,
I want **the app to display my carrier rates reliably without being blocked by hidden redundant settings I cannot see or control**,
so that **my checkout doesn't mysteriously break with no visible rates, I don't lose sales because customers can't see shipping options, and I don't have to open support tickets to fix configuration problems I didn't create and cannot diagnose myself**.

## Acceptance Criteria (Simple)

- [ ] Hidden redundant rate display setting is removed from WooCommerce plugin codebase
- [ ] All merchant configurations are migrated to use only the visible "RealTime Rates" setting
- [ ] Merchants affected by the hidden setting `false` bug have rates restored at checkout
- [ ] No future merchants can encounter this hidden setting failure mode
- [ ] Original ticket #383435 merchant verifies rates now display correctly

## Acceptance Criteria (Given/When/Then)

### Scenario: Redundant setting removed from codebase
- **Given** WooCommerce plugin has both a visible "RealTime Rates" setting and a hidden boolean setting controlling rate display
- **When** the redundant hidden setting is removed from the plugin
- **Then** only the visible "RealTime Rates" setting controls whether rates display at checkout
- **And** no code references the old hidden setting

### Scenario: Affected merchants migrated
- **Given** some WooCommerce stores have the hidden setting set to `false` (blocking rates despite RealTime Rates enabled)
- **When** the migration runs
- **Then** all affected stores have rates enabled based on their "RealTime Rates" configuration
- **And** checkout rates display correctly for previously-broken stores

### Scenario: Prevent future failures
- **Given** new WooCommerce stores install the plugin
- **When** they enable RealTime Rates
- **Then** rates display at checkout immediately with no hidden settings interference
- **And** no future merchants can encounter the "rates not showing despite setting enabled" issue from hidden config
