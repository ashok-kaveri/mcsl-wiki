## User Story

As a **high-volume German merchant shipping 500 DHL Paket labels per month who received a third-party API deprecation notice with a hard May 31 deadline**,
I want **proactive communication from StorePep when external carrier APIs change, with clear information about whether my label generation will be affected and what action I need to take**,
so that **I don't experience four days of panic thinking my entire shipping operation will stop, I don't waste L3 engineering time on false-alarm escalations, and I have confidence that the app will continue working even when carriers change their APIs**.

## Acceptance Criteria (Simple)

- [ ] Customer is notified when a carrier announces API changes, with clear impact assessment (affected / not affected)
- [ ] Support team has immediate access to which API version each carrier integration uses (documented in carrier config pages or admin panel)
- [ ] When a customer reports carrier API deprecation concerns, support can verify the actual API version in use within one response (not 4 days)
- [ ] DHL REST API migration is completed and stable for this customer's 500+ monthly labels
- [ ] Internal knowledge base documents which carriers use which API versions to prevent future confusion

## Acceptance Criteria (Given/When/Then)

### Scenario: Customer receives carrier API deprecation notice from third-party

- **Given** a customer using DHL Paket receives a deprecation notice from DHL about SOAP API v2
- **When** the customer opens a support ticket asking if they're affected
- **Then** support can immediately verify the app uses SOAP API v3 (not v2) and respond within 4 hours that the customer is not affected
- **And** support includes a timeline for the planned REST API migration as proactive information

### Scenario: Proactive communication for actual API migrations

- **Given** StorePep decides to migrate a carrier integration to a new API version
- **When** the migration is scheduled and tested
- **Then** affected customers receive advance notification with clear timeline, expected downtime (if any), and action items
- **And** support team is briefed on expected customer questions before the migration goes live

### Scenario: REST API migration prevents future SOAP deprecation concerns

- **Given** this customer has been migrated to DHL REST API
- **When** DHL announces future SOAP v3 deprecation or changes
- **Then** the customer's label generation continues uninterrupted because they're already on the modern API
- **And** the customer does not need to open another support ticket about DHL API changes
