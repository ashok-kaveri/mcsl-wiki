## User Story

As a **PluginHive product/engineering team committing to a paid custom carrier integration for a 10k-label/month Enterprise customer**,
I want **a real dev scope and time estimate for the Delivro (DispatchScience API) integration before we bill $200/month**,

- [ ] A dev scoping document exists for Delivro (DispatchScience API) covering: auth, rates, label, tracking, cancellation, manifest, returns — with effort estimate per surface

## Acceptance Criteria (Given/When/Then)

### Scenario 1: Scope doc delivered to Vape Wolinak
- **Given** the team has received DispatchScience portal credentials and the customer contact
- **When** dev scoping is complete
- **Then** a scope doc with effort estimate and target window is attached to ticket #381046 and shared with Cédric

- **Given** PluginHive's existing carrier integrations share a common abstraction
- **When** Delivro is implemented

- **Given** another merchant requests a paid custom carrier integration in future
- **When** the product team evaluates the request
- **Then** there is a documented scoping checklist (API surface, commercial pricing, staging creds, target timeline) that yields a consistent answer in days, not weeks
- **And** no paid integration quote is ever sent before the dev scope exists
