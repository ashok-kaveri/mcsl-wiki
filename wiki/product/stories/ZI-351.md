## User Story

As a **web development agency on the $29 plan building WooCommerce stores for multiple clients, managing 5+ active projects at once**,
I want **comprehensive multi-site setup documentation that answers: (1) Do I need separate EasyPost accounts per client? (2) How do I configure plugin per site while sharing one EasyPost account? (3) How do I keep client credentials isolated? (4) What's the best practice for handoff to client post-launch?**,
so that **I don't open a support ticket for questions that should be in a "Developer/Agency Setup Guide," can confidently quote accurate timelines to clients (30 minutes per site vs 2 hours figuring it out), and don't accidentally configure client A's EasyPost key on client B's site causing shipping cost billing errors**.

---

## Acceptance Criteria (Simple)

- [ ] Documentation page: "Agency & Developer Guide: Managing Multiple Client Sites" published in help center
- [ ] Guide covers: EasyPost account setup (1 account vs multiple), plugin installation per site, credential isolation best practices, post-launch handoff checklist
- [ ] During plugin installation: if developer email domain detected (e.g., @agency.com), show banner: "Managing multiple client sites? Read our Agency Setup Guide"
- [ ] Guide includes example: "Scenario: You manage 5 client stores. Use 1 EasyPost Production account + 1 Test account. Each client site gets its own plugin installation with the same EasyPost API key."
- [ ] FAQ answers: "Can I use one EasyPost account for all clients?" (Yes), "How do I bill shipping costs per client?" (EasyPost dashboard shows per-site usage)

## Acceptance Criteria (Given/When/Then)

### Scenario: Developer searches help docs for multi-site setup
- **Given** developer managing multiple client sites needs setup guidance
- **When** developer searches help center for "multiple sites" or "agency setup"
- **Then** top result is "Agency & Developer Guide: Managing Multiple Client Sites"
- **And** guide clearly states: "You can use ONE EasyPost account for all client sites. Install plugin separately on each site."

### Scenario: Developer installs plugin on 2nd client site
- **Given** developer already installed plugin on client A's site with EasyPost account
- **When** developer installs plugin on client B's site
- **Then** during setup, plugin detects same email used on another site and shows tip: "Managing multiple sites? You can use the same EasyPost API key across all client sites. Each site tracks shipping costs separately in EasyPost dashboard."

### Scenario: Developer wants to hand off site to client post-launch
- **Given** developer configured plugin with agency's EasyPost account
- **And** client wants to take over shipping management
- **When** developer views "Agency Setup Guide" → Post-Launch Handoff section
- **Then** guide provides checklist: "(1) Create new EasyPost account for client, (2) Update API key in plugin settings, (3) Test label generation, (4) Transfer plugin license to client email"
