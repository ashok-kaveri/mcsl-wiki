## User Story

As a **high-volume WooCommerce shipper processing ~10,000 labels/month with multi-item orders containing multiple products packed into a single box**,
I want **the Delivro integration to consolidate all products in an order into one shipment and generate exactly one label per package, respecting my WooCommerce packing configuration**,
so that **I can actually use the plugin for my Delivro shipments instead of being blocked by 30 duplicate labels per order that waste carrier credits, confuse warehouse staff, and force me to manually void each one**.

---

## Acceptance Criteria (Simple)

- [ ] A Delivro order with multiple products (e.g., 5 products, 15 qty) packed into a single package generates exactly one shipping label
- [ ] The label payload sent to the Dispatch Science API contains the consolidated weight and dimensions of the entire package, not individual product weights
- [ ] Existing packing configurations (box definitions, weight rules) in the WooCommerce plugin are respected by the Delivro label generation flow
- [ ] Multi-package orders (where the merchant explicitly configures multiple boxes) generate one label per configured package, not one label per product line item
- [ ] Previously generated duplicate labels for affected orders can be voided in bulk without requiring individual cancellation on the Delivro side

---

## Acceptance Criteria (Given/When/Then)

### Scenario: Single-package multi-item order generates one label
- **Given** a WooCommerce order containing 5 different products with a total of 15 quantity units, and the merchant's packing rules consolidate all items into a single 20lb package
- **When** the merchant generates a Delivro shipping label for this order through the WSS plugin
- **Then** exactly one label is created with the consolidated package weight (20lb) and dimensions, and the Dispatch Science API receives a single shipment request rather than 15 separate requests

### Scenario: Multi-package order generates correct number of labels
- **Given** a WooCommerce order with items that the merchant's packing configuration splits across 3 separate packages (e.g., fragile items separated from heavy items)
- **When** the merchant generates Delivro shipping labels for this order
- **Then** exactly 3 labels are created (one per configured package), each with its respective weight and dimensions, and no duplicate labels are generated for individual product line items within each package

### Scenario: Batch label generation respects packing consolidation across multiple orders
- **Given** a batch of 10 WooCommerce orders selected for Delivro label generation, each containing between 1 and 8 products packed into single packages
- **When** the merchant triggers batch label generation for all 10 orders simultaneously
- **Then** exactly 10 labels are generated (one per order), the total label count matches the total package count across all orders, and no order produces more labels than its configured package count
