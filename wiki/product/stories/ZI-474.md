## User Story

As a **BigCommerce merchant using FedEx who has been waiting 7 months for a delivery date format enhancement that was announced as complete but is still broken**,
I want **estimated delivery dates to display as actual dates (e.g., "Arrives March 25, 2026") instead of relative days ("Arrives in 3 days") at checkout**,
so that **my customers can make informed shipping decisions based on specific delivery dates rather than doing mental math to figure out when "3 days" actually means, especially important for time-sensitive gifts or events where customers need to know the exact date, not a relative count**.

## Acceptance Criteria (Given/When/Then)

### Scenario: Delivery date displays correctly at checkout
- **Given** a BigCommerce store is using Ship, Rate & Track for FedEx app
- **And** the store has delivery date display enabled
- **When** a customer selects a FedEx shipping option at checkout
- **Then** the estimated delivery shows as "Estimated delivery: March 25, 2026" (or configured date format)
- **And** NOT "Estimated delivery: in 3 days"

### Scenario: Feature actually works (not just announced as complete)
- **Given** development team announced on Mar 11 "we have enhanced the app to display delivery date along with service name"
- **And** support testing on Mar 23 found it still showing days instead of dates
- **When** QA verifies this fix
- **Then** testing must be done on actual BigCommerce storefront checkout, not just API responses or admin UI
- **And** test with multiple FedEx services to ensure all show dates consistently

### Scenario: Date format persists across app updates
- **Given** the delivery date feature is working correctly
- **When** future app releases are deployed
- **Then** automated regression test verifies delivery dates still display (not reverted to day counts)
- **And** test covers BigCommerce-specific rendering quirks
