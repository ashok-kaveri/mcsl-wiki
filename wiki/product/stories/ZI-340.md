## User Story

As a **Shopify development agency building stores for clients that require multi-carrier shipping (FedEx + UPS) before launch, managing multiple client projects with potential to bring more customers if successful**,
I want **at least one working UPS integration path when EasyPost blocks my account creation, whether that's a direct UPS API connection, an alternative aggregator (Shippo, ShipStation, Stamps.com), or clear documentation on why I was blocked and how to appeal**,
so that **I can complete the core shipping requirements my client paid for without telling them "we can't offer UPS rates because a third-party service won't let us create an account," and don't have to abandon 80% of a completed setup to find a different app that supports direct UPS integration**.

---

## Acceptance Criteria (Simple)

- [ ] Multi Carrier app supports at least 2 UPS integration paths: EasyPost + one alternative (direct UPS API, Shippo, ShipStation, or Stamps.com)
- [ ] If customer's EasyPost account creation is blocked, app displays actionable error: "EasyPost unavailable. Try alternative: [Configure Direct UPS] or [Use Shippo]"
- [ ] Alternative UPS integration supports same features as EasyPost (discounted rates, box packing, live rates at checkout)
- [ ] Documentation clearly lists UPS integration options with comparison table (EasyPost vs Direct vs Shippo) showing features, pricing, approval requirements
- [ ] If merchant must contact UPS directly for API access, app provides step-by-step guide with expected timeline (e.g., "UPS Developer Access takes 3-5 business days")

---

## Acceptance Criteria (Given/When/Then)

### Scenario: EasyPost blocks UPS account creation
- **Given** merchant has installed Multi Carrier app and requires UPS integration
- **When** merchant attempts to create EasyPost UPS account and receives policy/compliance block
- **Then** app displays: "EasyPost account creation failed. Alternative: Configure direct UPS connection (requires UPS developer account). [Learn More] [Start UPS Direct Setup]"
- **And** "Learn More" link explains why EasyPost might block (business type, country, volume) and how to appeal if error seems incorrect

### Scenario: Merchant configures alternative UPS integration (direct or Shippo)
- **Given** merchant's EasyPost attempt was blocked
- **When** merchant selects "Configure Direct UPS" or "Use Shippo" option
- **Then** app guides merchant through alternative setup: credentials required, expected discounts, approval timeline
- **And** alternative integration supports box packing, One Rate + Ground services, live rates at checkout (same feature parity as EasyPost)

### Scenario: Merchant wants to know UPS options before starting setup
- **Given** merchant is planning multi-carrier shipping setup
- **When** merchant views Carrier Configuration page → Add UPS
- **Then** app displays comparison: "UPS via EasyPost (instant, aggregated rates) vs UPS Direct (3-5 day approval, your negotiated rates) vs UPS via Shippo (instant, alternative aggregator)"
- **And** merchant can choose their preferred path before hitting an error
