# Ticket #390510 — Batch creation delays and label state management bugs

- **Customer**: Love of India (loveofindia.myshopify.com)
- **Product**: Shopify Multi Carrier Shipping Label
- **Duration**: 2026-05-18 (ongoing, 17 days)

## Timeline & Key Phases

### Phase 1: L3 Diagnostic — Batch and label state management issues (2026-05-18)
- Internal investigation by Veena documented 4 distinct production issues affecting large-volume label creation workflows
- Issues span batch processing, label state persistence, printing accuracy, and carrier priority logic
- No customer response yet; ticket remains open awaiting engineering triage

## Open Issues

1. **Batch creation delay and misleading error message for large volumes** — When creating batches of 1000+ orders, batch creation takes extended time. System fails to redirect to Label Batch page (smaller batches redirect correctly per video evidence). During processing, app displays "Error in Batch Creation" message even when batch is processing successfully, confusing customer. Area: label-generation. (Comment #0)

2. **Failed labels incorrectly persisting as "Processing" in original batch after retry** — When customer retries failed orders with a different carrier (e.g., 2 failures out of 100 orders), those same orders continue appearing as "Processing" in the original failed batch. Orders simultaneously appear in the new retry batch. Retried carrier name gets updated in original batch, creating UI/state confusion. May have related printing inconsistencies. Area: label-generation. (Comment #0)

3. **Printing mismatch — "Not in Ship" orders generating valid labels** — Order #45279880 was excluded from label print (not in "Not in Ship" status per Deepak), yet label was successfully generated and order was fulfilled in Shopify. Tracking ID remains valid (Amazon: 369348516162). First observed instance; root cause unclear. Indicates inconsistency between order status checks and label generation logic. Area: label-generation. (Comment #0)

4. **Implement carrier priority fallback logic** — Customer workflow requires: (1) create Amazon labels; (2) switch to Delhivery, retry failures; (3) retry all Delhivery failures with Amazon; (4) attempt remaining failures with XpressBees. This is a workflow enhancement request from customer's operational needs. Area: carrier-config. (Comment #0)

## User Story

As a **high-volume Indian merchant processing 1000+ order batches daily across Amazon, Delhivery, and XpressBees — whose warehouse workflow is broken by false error messages on batch creation, zombie "Processing" entries after retry, excluded orders getting labels anyway, and no automated carrier fallback when one carrier fails**,
I want **batch status to report accurately, retry operations to leave clean state without carrier name corruption, label generation to enforce order status gates, and carrier fallback to cascade automatically through my priority sequence**,
so that **my warehouse team trusts the system output without manually verifying every batch, we stop accidentally fulfilling orders that should be held, and failed carrier labels automatically retry with the next carrier instead of requiring manual intervention on hundreds of orders per day**.

---

## Acceptance Criteria (Simple)

- [ ] Batch creation for 1000+ orders shows real-time progress — not false "Error in Batch Creation" message
- [ ] Failed-then-retried orders appear only in retry batch; original batch marks them "Failed → Retried in Batch #X" with original carrier name preserved
- [ ] Orders in "Not in Ship" status are blocked from label generation at the API level
- [ ] Carrier fallback sequence (Amazon → Delhivery → XpressBees) is configurable per store and auto-executes on failure
- [ ] Love of India (loveofindia.myshopify.com) confirms all four issues resolved in production

## Acceptance Criteria (Given/When/Then)

### Scenario: Large batch shows accurate status
- **Given** 1200 orders selected for batch creation
- **When** the merchant clicks "Create Batch"
- **Then** UI shows progress with order count (not "Error in Batch Creation")
- **And** page redirects to Label Batch view on completion

### Scenario: Retry creates clean state
- **Given** a batch of 100 orders where 3 failed with Amazon
- **When** the manager retries those 3 with Delhivery
- **Then** original batch shows "Failed → Retried" (not "Processing") with Amazon as original carrier
- **And** retry batch contains only the 3 orders with Delhivery

### Scenario: Excluded orders blocked
- **Given** order #45279880 has status "Not in Ship"
- **When** a batch including this order is processed
- **Then** the order is skipped with log entry "Excluded: status Not in Ship"
- **And** no label generated, no fulfillment triggered on Shopify
