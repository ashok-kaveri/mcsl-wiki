## Acceptance Criteria (Given/When/Then)

### Scenario: $9 plan merchant gets live tracking as advertised
- **Given** a merchant is on the $9 plan and the pricing page lists live tracking as included at that tier
- **When** a shipment is created and the carrier provides tracking updates
- **Then** the delivery status updates automatically in the MCSL app and syncs to the Shopify order timeline

### Scenario: Migrated merchant retains feature parity
- **Given** a merchant migrated from a legacy single-carrier app (e.g., PH Ship Rate & Track for FedEx) to the MCSL app
- **When** the merchant's new plan tier advertises the same features they had before (live tracking, label generation)
- **Then** all advertised features are active immediately without requiring manual enablement or L3 escalation

### Scenario: Pricing page and plan entitlements stay in sync
- **Given** engineering changes the feature set for any plan tier
- **When** the change is deployed
- **Then** the pricing page is updated in the same release, and no plan tier advertises a feature that is not programmatically enabled
