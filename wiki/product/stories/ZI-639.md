# Ticket #392155 — WooCommerce batch product import fails on stores with HTTP Basic Auth

- **Customer**: Shawn Quait (180drinks.ca — staging site: staging180drinksca.bigscoots-staging.com)
- **Product**: WooCommerce Shipping Services (WSS)
- **Duration**: 2026-05-26 to 2026-06-05 (11 days, resolved for customer)

## Timeline & Key Phases

### Phase 1: Staging Setup and Trial Extension (2026-05-26)
- Customer migrating WooCommerce site from 180drinks.ca to new hosting (BigScoots staging)
- Installed WSS plugin on staging, noticed 14-day trial expiring
- Canada Post carrier configured, 0 orders initially

### Phase 2: Product Import Failures and API Debugging (2026-05-27 to 2026-06-02)
- Products not importing into WSS plugin despite manual import attempts
- L3 (Veena) identified: staging site has HTTP Basic Authentication enabled but credentials not configured in plugin — fixed
- Customer updated plugin to latest version (2.1.8), import still failing
- L3 found API responses blank despite Postman returning data correctly
- IP whitelisting requested and completed by hosting (BigScoots)
- Multiple back-and-forth between customer, support, and hosting provider about WAF/auth blocking

### Phase 3: Root Cause Discovery — Batch Import Bug (2026-06-03)
- Athul (L3) investigated and found **code bug**: batch product import does NOT pass Basic Auth credentials
- Normal import correctly includes `store.basicAuth` check with username/password in API request
- Batch import path is missing this check entirely — API requests go out without auth headers, get rejected silently
- **Workaround applied**: disabled batch import for this customer, used normal import flow — all 815 products imported successfully

### Phase 4: Checkout Rates and Shipping Zone Configuration (2026-06-03 to 2026-06-05)
- Rates at checkout working after Ship From Address was configured
- Customer needed Ontario-only shipping restriction for alcohol products (licensing requirement)
- Resolved on a call: automation rules configured for alcohol shipping class → Ontario-only zone
- Ticket closed, customer satisfied

## Open Issues

1. **Batch product import missing Basic Auth credentials** — Normal product import passes `store.basicAuth` credentials in API headers, but batch import code path does not. Stores with HTTP Basic Authentication enabled silently fail batch imports. Workaround: disable batch import per-store. Blocked: engineering (code fix needed). Severity: medium (workaround exists but is per-customer manual intervention). Area: product-management. (Slack thread: #app-mcsl-wss-bss-mss-pss 2026-06-03, Athul's bug report)

## Resolved Issues

1. **Basic Auth not configured in plugin** — L3 added staging site's Basic Auth credentials to plugin settings. (Comment #20)
2. **Outdated plugin version** — Customer updated from old version to 2.1.8. (Comment #21)
3. **Rates failing at checkout** — Ship From Address was not configured; once set, rates returned correctly. (Comment #38)
4. **Alcohol shipping zone restriction** — Automation rules configured on call to restrict alcohol products to Ontario-only shipping. (Comment #55-56)

## User Story

As a **WooCommerce store owner who has HTTP Basic Authentication enabled on my site for security**,
I want **product imports to work regardless of whether they run in normal or batch mode**,
so that **my 800+ products import reliably without PluginHive support having to manually disable batch mode on my account, and every other store with Basic Auth doesn't hit the same silent failure**.

---

## Acceptance Criteria (Simple)

- [ ] Batch product import passes Basic Auth credentials (`store.basicAuth`, `store.username`, `store.password`) in API request headers
- [ ] Normal and batch import use the same auth logic — no divergence in credential handling
- [ ] Stores with Basic Auth enabled can import 800+ products via batch without manual intervention
- [ ] Stores without Basic Auth are unaffected (no regression)
- [ ] Existing workaround (batch import disabled per-store) can be reverted for affected customers

---

## Acceptance Criteria (Given/When/Then)

### Scenario: Batch import on store with Basic Auth
- **Given** a WooCommerce store has HTTP Basic Auth enabled and credentials configured in WSS plugin settings
- **When** batch product import is triggered
- **Then** API requests include Basic Auth headers and products import successfully

### Scenario: Batch import on store without Basic Auth
- **Given** a WooCommerce store does NOT have Basic Auth enabled
- **When** batch product import is triggered
- **Then** import works as before (no regression from auth changes)

### Scenario: Large catalog with Basic Auth
- **Given** a WooCommerce store with Basic Auth and 800+ products
- **When** batch product import runs
- **Then** all products import completely without timeout or silent failure
