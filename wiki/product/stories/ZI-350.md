## User Story

As an **India-based merchant on the $19 plan processing 184 orders, using FedEx for rate quotes only, who needs to migrate from SOAP to REST API but doesn't understand why support is asking for my FedEx account holder's first and last name**,
I want **a self-service FedEx REST migration wizard that auto-detects my FedEx account details from my existing SOAP credentials, or clearly explains upfront what personal information I'll need to provide and why FedEx REST API requires it (account verification)**,
so that **I don't get stuck waiting on a support call to manually collect information that should be automated, don't worry whether I'm giving the "right" name (is it my Shopify name? My FedEx account name? My legal business name?), and can complete the migration in 5 minutes instead of opening a WhatsApp thread and scheduling a call**.

---

## Acceptance Criteria (Simple)

- [ ] FedEx REST migration wizard pre-fills account holder name from existing FedEx SOAP account data (if available)
- [ ] If name can't be auto-detected: wizard explains "FedEx REST API requires the account holder's full name for authentication. This should match the name on your FedEx account."
- [ ] Wizard validates entered name against FedEx API before completing migration (rejects if name doesn't match account)
- [ ] If validation fails: error message shows "The name '[entered name]' doesn't match your FedEx account. Please enter the account holder's full name exactly as it appears on FedEx.com."
- [ ] Migration success email confirms: "FedEx REST migration complete. Your rate quotes now use the updated API. No further action needed."

---

## Acceptance Criteria (Given/When/Then)

### Scenario: Migration wizard auto-detects account name from SOAP credentials
- **Given** merchant has FedEx SOAP integration active with account holder name "Maria Pothys"
- **When** merchant initiates FedEx REST migration
- **Then** migration wizard displays: "Account Holder Name: Maria Pothys (detected from your existing FedEx account). [Confirm] [Edit]"
- **And** if merchant clicks "Confirm", migration proceeds without manual name entry

### Scenario: Migration wizard cannot auto-detect name, requests manual entry
- **Given** merchant's FedEx SOAP account doesn't include account holder name in API response
- **When** merchant initiates FedEx REST migration
- **Then** wizard displays form: "FedEx REST API requires your account holder's full name. Enter the name exactly as shown on your FedEx account. [First Name] [Last Name]. Why is this needed? FedEx uses this for account verification."
- **And** merchant enters name and clicks "Next"
- **Then** wizard validates name against FedEx API before proceeding

### Scenario: Name validation fails, merchant enters incorrect name
- **Given** merchant enters "John Smith" but FedEx account is under "Maria Pothys"
- **When** migration wizard validates name with FedEx API
- **Then** error displays: "The name 'John Smith' doesn't match your FedEx account. Please check FedEx.com → My Profile for the correct account holder name, then re-enter."
- **And** migration does not proceed until correct name is provided
