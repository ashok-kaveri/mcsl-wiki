## User Story

As a **merchant who cannot generate shipping labels today, with orders piling up waiting for fulfillment, who already checked my automation rules as support suggested in previous ticket #374249 but the issue persists**,
I want **label generation errors to show specific, actionable error messages ("Automation rule #3 failed: no carrier found for destination Canada" instead of generic "Unable to issue label"), automatic detection of automation rule conflicts when they're created (before they break label generation), and a "Skip automation rules for this order" emergency override button when I need to ship NOW**,
so that **I don't lose an entire day of fulfillment time waiting for support to diagnose a vague error, can immediately understand which automation rule is breaking label generation, and have a fallback to manually generate labels while support investigates instead of telling customers their orders are delayed because my shipping app is down**.

## Acceptance Criteria (Simple)

- [ ] Label generation errors show specific failure reason: "Automation rule '[Rule Name]' blocked label generation. Reason: [specific issue]"
- [ ] Error message includes troubleshooting: "To ship this order now: click 'Skip Automation Rules' below. To fix rule: go to Settings → Automation Rules → [Rule Name]."
- [ ] "Skip Automation Rules" button appears on error screen, generates label manually without automation logic applied
- [ ] When automation rule is saved: app validates rule logic and warns "This rule may prevent labels for orders to [Canada] because no carrier is configured for that destination"
- [ ] Support dashboard shows "Label Generation Failures" alert with: store, error reason, which automation rule failed, timestamp

## Acceptance Criteria (Given/When/Then)

### Scenario: Automation rule blocks label generation with specific error
- **Given** merchant has automation rule: "If destination = Canada, use carrier UPS"
- **And** UPS is not configured in the app
- **When** merchant attempts to generate label for Canadian order
- **Then** error displays: "Cannot generate label. Automation rule 'Canada Orders → UPS' requires UPS carrier, but UPS is not configured. [Configure UPS Now] or [Skip Automation Rules]"
- **And** merchant can click "Skip Automation Rules" to generate label with default carrier

### Scenario: Merchant uses emergency override to fulfill order immediately
- **Given** label generation is failing due to automation rule issue
- **And** merchant has orders waiting to ship
- **When** merchant clicks "Skip Automation Rules" on label generation screen
- **Then** label generates immediately using default carrier (ignoring all automation rules for this one order)
- **And** order notes log: "Label generated with automation rules skipped (emergency override by merchant)"

### Scenario: Automation rule validation prevents future label failures
- **Given** merchant creates automation rule: "If weight > 10lbs, use FedEx Freight"
- **And** FedEx Freight is not available in merchant's FedEx account
- **When** merchant clicks "Save Rule"
- **Then** validation warning displays: "Warning: FedEx Freight is not enabled on your FedEx account. Orders matching this rule will fail to generate labels. [Edit Rule] or [Save Anyway]"
