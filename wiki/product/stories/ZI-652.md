## User Story

As a **WooCommerce merchant shipping car stereo equipment from New Jersey to customers in Puerto Rico using USPS**,
I want **USPS rate requests to correctly treat Puerto Rico as a domestic US territory instead of passing country code "PR"**,
so that **my Puerto Rico customers see USPS Priority Mail rates at checkout and I don't lose sales to a shipping destination I should be able to serve domestically**.

---

## Acceptance Criteria (Simple)

- [ ] Rate requests for Puerto Rico addresses use domestic US addressing (no country code "PR") when calling the USPS REST API
- [ ] USPS Priority Mail and other domestic services return valid rates for Puerto Rico destinations
- [ ] The same territory normalization applies to other US territories (Guam, USVI, American Samoa, Marshall Islands, Northern Mariana Islands) that USPS treats as domestic
- [ ] WooCommerce shipping zones configured for Puerto Rico continue to trigger WSS rate calculation without manual workarounds
- [ ] The fix covers both USPS Ship and Stamps.com account types (per prior ticket #393254 precedent)

---

## Acceptance Criteria (Given/When/Then)

### Scenario: Puerto Rico address returns domestic USPS rates
- **Given** a WooCommerce customer enters a shipping address with country "PR" and a valid Puerto Rico zip code (e.g., 00901)
- **When** the rate shopping engine sends the rate request to USPS REST API
- **Then** the request treats the destination as domestic US, and USPS returns Priority Mail, Ground Advantage, and other domestic service rates

### Scenario: Other US territories also normalized for USPS
- **Given** a WooCommerce customer enters a shipping address for Guam (GU), US Virgin Islands (VI), or American Samoa (AS)
- **When** the rate shopping engine processes the USPS rate request for that territory
- **Then** the destination is treated as domestic US and valid USPS domestic rates are returned without "Bad Request" errors

### Scenario: Non-territory international addresses remain unaffected
- **Given** a WooCommerce customer enters a shipping address for Canada (CA) or Mexico (MX)
- **When** the rate shopping engine sends the rate request to USPS REST API
- **Then** the request correctly uses international addressing and returns international service rates, confirming no regression
