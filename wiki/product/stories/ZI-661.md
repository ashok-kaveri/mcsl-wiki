## Timeline & Key Phases

### Phase 1: Initial report and investigation (2026-06-12)
- Customer reported PostNord "Mypack Collect" service unavailable; order #172 shows "FAILURE Contact support" with empty request body (Comment #1)

- L3 applied the workaround on the customer's store; orders 171 and 172 now in Processing status (Comment #25)
- Customer acknowledged the update (Comment #26)

## User Story

As a **B2B merchant on BigCommerce who invoices customers separately and therefore has zero-value order totals in my store**,
I want **the app to handle currency conversion gracefully when the order total is zero, without breaking the rate or label generation flow**,
so that **I can generate PostNord shipping labels for all my international orders without manually switching currency settings as a workaround every time I process shipments**.

---

## Acceptance Criteria (Simple)

- [ ] When preferred shipment currency differs from order currency and order total is 0, the currency conversion logic returns 0 instead of erroring with a division-by-zero or invalid conversion
- [ ] Rate fetching returns valid PostNord service options for orders with zero total when currency conversion is configured
- [ ] Label generation succeeds for zero-total orders regardless of the preferred shipment currency setting
- [ ] The fix handles all currency pairs (not just SEK-to-EUR) where order total is 0 and a conversion is attempted
- [ ] Orders that previously failed due to this bug can be retried successfully without manual workaround after the fix is deployed

---

## Acceptance Criteria (Given/When/Then)

### Scenario: Zero-total order with cross-currency conversion
- **Given** a BigCommerce store has preferred shipment currency set to EUR and an order with total 0 SEK (B2B separate invoicing)
- **When** the merchant opens the order summary page and requests PostNord shipping rates
- **Then** the app converts 0 SEK to 0 EUR without error and returns valid PostNord service options with their standard rates

### Scenario: Label generation succeeds for zero-total cross-currency order
- **Given** a merchant has a zero-total order in SEK with preferred shipment currency set to EUR and has selected a PostNord Mypack Collect service
- **When** they click Generate Label on the order
- **Then** the label is generated successfully with correct sender/receiver details, and the shipment value on the label and customs declaration reflects 0 in the target currency

### Scenario: Non-zero orders unaffected by the fix
- **Given** a BigCommerce store has preferred shipment currency set to EUR and an order with total 150 SEK
- **When** the merchant requests rates and generates a label for this order
- **Then** the currency conversion calculates the correct EUR equivalent of 150 SEK and the label generation completes with the converted amount on the customs declaration
