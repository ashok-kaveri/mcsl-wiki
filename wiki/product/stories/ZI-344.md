## User Story

As a **Hong Kong merchant on the $29 plan generating 202 labels in April alone, managing a product catalog via CSV imports because I have hundreds of SKUs with dimension and customs data required for international shipping (FedEx, HK Post, UPS)**,
I want **CSV product import to only update/add the products explicitly listed in my CSV file, without touching or deleting data for existing products that aren't in the file**,
so that **I don't lose hours of work configuring dimensions and customs descriptions for hundreds of existing products every time I import 10 new styles, avoid label generation failures for orders containing the wiped products (wrong dimensions = wrong rates = carrier billing disputes), and don't have to choose between never adding new products or manually re-entering existing product data after every import**.

---

## Acceptance Criteria (Simple)

- [ ] CSV import uses "upsert" logic: update products in CSV, leave all other products untouched
- [ ] Import preview shows: "X products will be updated, Y products will be added, 0 products will be deleted"
- [ ] If CSV is missing a product that exists in app: app keeps existing data (doesn't delete it)
- [ ] Import confirmation screen lists products that will lose data if import proceeds (shows before/after comparison)
- [ ] App logs all product data changes from CSV import with rollback option (restore previous dimensions/customs data for last 7 days)

---

## Acceptance Criteria (Given/When/Then)

### Scenario: Merchant imports CSV with only new products
- **Given** app has 100 products with dimension and customs data configured
- **And** merchant creates CSV with 10 new products only (doesn't include existing 100)
- **When** merchant uploads CSV and clicks "Import Products"
- **Then** app adds 10 new products without modifying or deleting data for existing 100 products
- **And** existing products retain their dimensions, customs descriptions, and all configured data

### Scenario: Merchant imports CSV that updates some existing products
- **Given** app has products A, B, C with dimensions configured
- **And** merchant creates CSV with products A (updated dimensions) and D (new product)
- **When** merchant imports CSV
- **Then** product A's dimensions are updated per CSV
- **And** products B and C retain their existing dimensions (unchanged)
- **And** product D is added with CSV dimensions

### Scenario: Import preview warns about potential data loss
- **Given** merchant is about to import CSV with full product replacement (old import logic)
- **When** merchant clicks "Import Products"
- **Then** preview screen shows: "WARNING: 50 existing products are NOT in this CSV. If you proceed, they will lose their dimension data. [Cancel] [Proceed Anyway] [Download those 50 products to add to CSV]"
- **And** merchant can download existing product data to merge with new CSV before importing
