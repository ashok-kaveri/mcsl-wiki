## User Story

As a **Shopify store manager who adjusts product weights when updating inventory** (e.g., packaging changes, reweighing items), I want **the shipping label system to immediately use the updated weight**, so that **I don't generate labels with stale weight data that causes carriers to reject shipments or charge unexpected weight-based surcharges**.

---

## Acceptance Criteria (Given/When/Then)

### Scenario: Merchant updates product weight in Shopify

- **Given** I edit a product's weight in Shopify Admin (e.g., 2 lbs → 2.5 lbs)
- **When** I generate a shipping label for an order containing that product
- **Then** the label uses the updated weight (2.5 lbs)
- **And** carrier rates reflect the new weight (not the old 2 lb rate)

### Scenario: Bulk weight updates sync across multiple products

- **Given** I update 10 product weights in Shopify (e.g., new packaging)
- **When** I generate labels for orders containing these products within the same day
- **Then** all labels use the updated weights
- **And** the app doesn't cache stale product data from hours earlier

### Scenario: No stale weight data in label generation

- **Given** any product weight is updated
- **When** labels are generated any time after the update
- **Then** the system never uses the old weight
- **And** if there's a sync delay, the system alerts me ("product data sync in progress")
