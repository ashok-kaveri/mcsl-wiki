## User Story

As a **US titanium-racing-parts seller on the $49 Premium plan with 485 orders** whose warehouse occasionally edits a shipping address right before label print and has to remember to re-enter it inside WSS, I want **WooCommerce order edits (shipping address, line items) to automatically propagate into WSS after the order is already imported**, so that **we stop shipping to stale addresses, stop doing customer-service work on returned mis-deliveries, and stop relying on operators to remember to update two systems for one order change**.

---

## Acceptance Criteria (Simple)

- [ ] When the shipping address is edited on a WooCommerce order that's already in WSS, the change syncs into WSS within minutes without manual action
- [ ] Line-item changes (added/removed/qty-changed) on an imported order also sync into WSS
- [ ] A visible sync-event log on the order in WSS shows when the last update was received and what changed
- [ ] Sync respects label-generated state — if a label has already been printed, the update surfaces as a warning rather than silently overwriting
- [ ] Customer Josef Bosnyak (RaceTech Titanium) can volunteer-test the feature as offered in Comment #11

---

## Acceptance Criteria (Given/When/Then)

### Scenario: Address change before label print propagates automatically

- **Given** a WooCommerce order already imported into WSS with shipping Address A
- **When** the merchant edits the shipping address in WooCommerce Order Details to Address B and saves
- **Then** WSS reflects Address B within 60 seconds, with no manual action by the operator
- **And** if the operator generates a label after the sync, it uses Address B

### Scenario: Edit after label print surfaces a clear warning

- **Given** an order in WSS where a label has already been generated using Address A
- **When** the merchant edits the shipping address in WooCommerce to Address B
- **Then** WSS displays a 'shipping address changed after label generation — verify before shipping' warning on the order
- **And** WSS does not silently overwrite the printed-label record

### Scenario: Line-item change syncs and recomputes packing

- **Given** an imported order with 3 line items
- **When** the merchant removes one item in WooCommerce
- **Then** WSS reflects the 2-item state within 60 seconds and re-quotes the rate
- **And** the operator sees the change in the order's sync-event log
