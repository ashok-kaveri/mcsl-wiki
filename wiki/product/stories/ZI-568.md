## Open Issues

1. **Address Line Duplication on FedEx Regional Economy labels** — Street name/address line 1 printed twice on labels, address correction window, and commercial invoices when using FedEx Regional Economy service after REST migration. FedEx International Priority fix was applied but Regional Economy duplication persists. Blocked: Dev team. Severity: High (affects label accuracy; data integrity). Area: label-generation. (Comments #27-35, #45-50)

2. **Automation rules service selection logic broken for FedEx REST** — When customer switches FedEx from SOAP to REST, automation rules that previously defaulted to FedEx now select cheapest available service instead (e.g., DHL if cheaper). Customer has 3 rules: Rule 1 (FedEx Int'l Priority default), Rule 2 (FedEx Regional Economy), Rule 3 (DHL). With old SOAP account, Rule 1 always picked FedEx Int'l Priority. With new REST account, rule selection logic is inverted — service selection algorithm differs between SOAP and REST integrations. Root cause identified: service selection logic does not account for rule-specific carrier preference when REST API is active. Blocked: Dev team. Severity: Critical (breaks order automation, forces manual intervention). Area: carrier-config. (Comments #27-28, #35, #51-54)

## Resolved Issues

1. **FedEx REST Account registration failure** — Initial problem was plan limitation (Basic Plan only allows 1 carrier, customer had 3). Resolved 2026-05-18 by plan upgrade and removal of unused SOAP account. Successfully migrated both stores to FedEx REST. (Comment #23, #25-26)

2. **Address Line Duplication on FedEx International Priority** — Duplicated street name appearing on labels and documents after REST migration. Fixed by server-side change on 2026-05-20. Customer confirmed fixed on 2026-05-21. (Comments #33-34, #35)

## User Story

As a **Shopify merchant running two international stores from Lithuania who migrated to FedEx REST and now sees the street address printed twice on every FedEx Regional Economy label — creating delivery confusion and potential carrier rejections on 85+ monthly shipments — while the same issue was already fixed for International Priority on 2026-05-20**,
I want **FedEx Regional Economy labels to print each address line exactly once, matching the address formatting already working correctly on International Priority labels after the earlier fix**,
so that **my labels are accurate, carriers don't flag or reject shipments for malformed addresses, and I don't have to visually inspect every Regional Economy label before handing it to the courier**.

## Acceptance Criteria (Given/When/Then)

### Scenario: Regional Economy label prints clean address
- **Given** an order with ship-to "123 Main Street, Vilnius, Lithuania" on a FedEx REST account
- **When** a Regional Economy label is generated
- **Then** the label PDF shows "123 Main Street" once in the recipient address block
- **And** no address field is duplicated anywhere on the label

### Scenario: Parity with International Priority fix
- **Given** the same order shipped via International Priority (already fixed)
- **When** the merchant switches to Regional Economy and regenerates
- **Then** address formatting is identical between both services
- **And** no regression in the International Priority label

### Scenario: Multi-line address with apartment/suite
- **Given** a ship-to address "456 Oak Ave, Apt 12B, Kaunas, Lithuania"
- **When** a Regional Economy label is generated
- **Then** "456 Oak Ave" and "Apt 12B" each appear once on separate lines
- **And** the full address renders correctly within the label's address block
