## User Story

As a **European Shopify merchant who ships domestically and cross-border through GLS — one of the largest parcel carriers in Europe covering 40+ countries — and who currently generates GLS labels on a separate portal because MCSL doesn't support GLS despite an active partnership discussion with API credentials already shared**,
I want **GLS available as a carrier in MCSL through a phased integration (Phase 1: ShipIT API for centralized coverage, Phase 2: MyGLS for regional markets like Czech Republic and Ireland)**,
so that **I manage all carrier labels in one app, get the same automation and batch processing for GLS that I have for FedEx/DHL/UPS, and European customers who rely on GLS for domestic parcel delivery aren't excluded from my shipping optimization**.

## Acceptance Criteria (Given/When/Then)

### Scenario: Merchant connects GLS via ShipIT
- **Given** a European merchant with GLS ShipIT account credentials
- **When** they enter credentials in MCSL carrier settings
- **Then** GLS appears as available carrier with supported services
- **And** a test API call confirms connectivity

### Scenario: Cross-border label generation
- **Given** an order shipping from Czech Republic to Germany via GLS
- **When** the merchant selects GLS and generates a label
- **Then** a valid GLS label PDF with tracking number is returned
- **And** tracking syncs via the already-integrated Track & Trace API

### Scenario: Rate comparison across carriers
- **Given** GLS, DHL, and FedEx all enabled for European routes
- **When** rate shopping runs for a Prague → Berlin parcel
- **Then** all three carriers return quotes ranked by price and transit time
- **And** GLS rates reflect the merchant's ShipIT contract pricing
