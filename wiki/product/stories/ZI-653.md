## User Story

As a **Shopify merchant whose customers frequently request shipments billed to their own UPS accounts (about 35 of every 100 orders)**,
I want **to enter a customer's UPS account number and billing postal code on a per-order basis through the bulk actions workflow (similar to how FedEx Special Services already works)**,
so that **I can generate labels charged to my customer's UPS account without creating individual automation rules for each customer, which currently forces me to consider switching to ShipStation**.

---

## Acceptance Criteria (Simple)

- [ ] Bulk Actions menu includes a "UPS Third-Party Billing" option that accepts UPS account number and billing zip/postal code per selected order
- [ ] Labels generated with third-party billing charge the specified UPS account, not the merchant's account
- [ ] The third-party billing fields are optional — orders without them default to the merchant's UPS account as today
- [ ] The workflow mirrors the existing FedEx Special Services pattern (More Actions > entry form > apply to selected orders)
- [ ] Third-party billing data persists on the order record so re-prints or rate re-fetches use the same billing account

---

## Acceptance Criteria (Given/When/Then)

### Scenario: Merchant applies third-party UPS billing via bulk actions
- **Given** a merchant selects one or more orders in the MCSL order grid that need to ship on a customer's UPS account
- **When** they choose "UPS Third-Party Billing" from Bulk Actions and enter the customer's UPS account number and billing postal code
- **Then** the label generation request includes the third-party billing payload, and UPS charges the customer's account instead of the merchant's

### Scenario: Orders without third-party billing use merchant account
- **Given** a merchant selects orders in the MCSL order grid that should ship on the merchant's own UPS account
- **When** they generate labels without entering third-party billing details
- **Then** labels are billed to the merchant's registered UPS account, confirming no change to default behavior

### Scenario: Third-party billing data retained for label reprint
- **Given** a merchant previously generated a UPS label with third-party billing for a specific order
- **When** the merchant reprints or voids and regenerates the label for that order
- **Then** the third-party billing account number and postal code are pre-populated from the original entry, not requiring re-entry
