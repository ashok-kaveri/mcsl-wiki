## Acceptance Criteria (Given/When/Then)

### Scenario: Direct fix for reported issue
- **Given** the merchant has the exact configuration described in ticket #378513
- **When** the merchant performs the action that previously triggered the issue
- **Then** the action completes successfully without errors
- **And** the result matches what the merchant expected

### Scenario: Fix holds for similar configurations
- **Given** a different merchant has a similar but not identical setup (different carrier account, different product mix)
- **When** they perform the same action
- **Then** the fix still applies and the action succeeds
- **And** no edge case triggers the original failure

### Scenario: Failure is self-diagnosable
- **Given** a future merchant encounters a related issue that the fix doesn't cover
- **When** the error occurs
- **Then** the error message identifies the specific field, carrier, or configuration causing the problem
- **And** the merchant can resolve it without opening a support ticket
